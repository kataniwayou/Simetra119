3. At the TOP of the loop body, add Gate 1 backoff check:
```csharp
if (!_preferredLeaderService.IsPreferredPod && _stampReader.IsPreferredStampFresh)
{
    _logger.LogDebug(
        "Preferred pod is alive — delaying election attempt for {DurationSeconds}s",
        _leaseOptions.DurationSeconds);
    await Task.Delay(TimeSpan.FromSeconds(_leaseOptions.DurationSeconds), stoppingToken);
    continue;  // re-evaluate at top of loop
}
```

Key behavior:
- When `IsPreferredPod` is true: skip the backoff entirely (preferred pod never delays)
- When `IsPreferredStampFresh` is false: skip the backoff (no preferred pod alive)
- Only when BOTH `!IsPreferredPod` AND `IsPreferredStampFresh`: delay by DurationSeconds then `continue` to re-check

4. After the backoff check, create the inner CTS and run the election:
```csharp
using var innerCts = CancellationTokenSource.CreateLinkedTokenSource(stoppingToken);
_innerCts = innerCts;

try
{
    await elector.RunAndTryToHoldLeadershipForeverAsync(innerCts.Token);
}
catch (OperationCanceledException) when (stoppingToken.IsCancellationRequested)
{
    break;  // Outer shutdown — exit loop cleanly
}
catch (OperationCanceledException)
{
    // Inner cancel (Phase 88 voluntary yield) — loop continues, re-evaluate backoff
}
finally
{
    _innerCts = null;
}
```

5. The `Task.Delay` for backoff uses `stoppingToken` (NOT `_innerCts.Token`) because _innerCts does not exist yet at that point.

6. The `OperationCanceledException` catch differentiates outer shutdown (stoppingToken cancelled -> break) from inner cancel (Phase 88 -> continue loop).

7. Do NOT modify OnStoppedLeading — it must remain `_isLeader = false` only. This is the idempotency requirement.

8. Do NOT modify StopAsync — _innerCts is linked to stoppingToken so it auto-cancels. The `using` block handles disposal.
