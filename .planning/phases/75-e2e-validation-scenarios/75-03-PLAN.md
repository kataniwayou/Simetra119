Create `tests/e2e/scenarios/112-tvm06-all-instances.sh` — verifies tenant metrics export on all pod instances (not just leader), while snmp_gauge/snmp_info remain leader-gated.

**Key architecture point:** The `SnmpCollector.Tenant` meter is NOT gated by `MetricRoleGatedExporter`. It exports on ALL instances (leader + followers). The `SnmpCollector.Leader` meter is gated — only the leader exports snmp_gauge/snmp_info.

**Scenario structure:**
1. Standard setup: save ConfigMap, apply `tenant-cfg06-pss-single.yaml`, wait for reload.
2. Prime T2 OIDs: `sim_set_oid "5.1" "10"`, `"5.2" "1"`, `"5.3" "1"`.
3. Sleep 8s for grace.
4. Wait for at least one healthy evaluation cycle — `poll_until_log 30 1 "e2e-pss-tenant.*tier=3" 15`.
5. Sleep 5s for metrics to propagate to Prometheus.

**Preflight check (from scenario 106 pattern):**
Verify `k8s_pod_name` label exists on `tenant_state`:
```bash
VERIFY_LABEL=$(query_prometheus 'tenant_state{tenant_id="e2e-pss-tenant"}' | jq -r '.data.result[0].metric.k8s_pod_name // ""')
if [ -z "$VERIFY_LABEL" ]; then
    record_fail "TVM-06: preflight" "k8s_pod_name label not present on tenant_state"
    # cleanup and return
fi
```

**Get all pod names:**
```bash
POD_NAMES=$(kubectl get pods -n simetra -l app=snmp-collector \
    -o jsonpath='{range .items[*]}{.metadata.name}{"\n"}{end}')
```

**Sub-assertions:**

TVM-06A: tenant_state present on EVERY pod (all-instances export)
```bash
ALL_PODS_HAVE_TENANT=true
for POD in $POD_NAMES; do
    [ -z "$POD" ] && continue
    COUNT=$(query_prometheus "tenant_state{k8s_pod_name=\"${POD}\",tenant_id=\"e2e-pss-tenant\"}" \
        | jq -r '.data.result | length')
    if [ "$COUNT" -gt 0 ]; then
        record_pass "TVM-06A: tenant_state present on pod ${POD}" "k8s_pod_name=${POD} series_count=${COUNT}"
    else
        record_fail "TVM-06A: tenant_state present on pod ${POD}" "k8s_pod_name=${POD} series_count=0"
        ALL_PODS_HAVE_TENANT=false
    fi
done
```

TVM-06B: snmp_gauge ABSENT on follower pods (leader-gated meter confirmed)
Follow the pattern from scenario 106: identify follower pods (snmp_gauge series count == 0 for that pod), verify at least one follower exists.
```bash
FOLLOWER_COUNT=0
for POD in $POD_NAMES; do
    [ -z "$POD" ] && continue
    GAUGE_COUNT=$(query_prometheus "snmp_gauge{k8s_pod_name=\"${POD}\"}" | jq -r '.data.result | length')
    TENANT_COUNT=$(query_prometheus "tenant_state{k8s_pod_name=\"${POD}\",tenant_id=\"e2e-pss-tenant\"}" | jq -r '.data.result | length')
    if [ "$GAUGE_COUNT" -eq 0 ] && [ "$TENANT_COUNT" -gt 0 ]; then
        FOLLOWER_COUNT=$((FOLLOWER_COUNT + 1))
        record_pass "TVM-06B: follower ${POD} has tenant_state but no snmp_gauge" "snmp_gauge=0 tenant_state=${TENANT_COUNT}"
    fi
done
if [ "$FOLLOWER_COUNT" -eq 0 ]; then
    record_fail "TVM-06B: at least one follower exports tenant metrics without snmp_gauge" "follower_count=0"
fi
```
