**Class structure:**
- `namespace SnmpCollector.Services;`
- `public sealed class ChannelConsumerService : BackgroundService`
- Constructor dependencies (inject via DI):
  - `IDeviceChannelManager channelManager`
  - `ISender sender` — inject ISender (NOT IMediator, NOT IPublisher). ISender is the correct interface for IRequest<Unit> dispatch. MediatR auto-registers ISender.
  - `PipelineMetricService pipelineMetrics`
  - `ILogger<ChannelConsumerService> logger`
- Store all as private readonly fields

**CRITICAL: Use ISender.Send(), NOT IPublisher.Publish(). SnmpOidReceived implements IRequest<Unit>, not INotification. IPublisher.Publish bypasses the entire behavior pipeline (Logging, Exception, Validation, OidResolution would NOT run).**

**ExecuteAsync(CancellationToken stoppingToken):**
```csharp
protected override async Task ExecuteAsync(CancellationToken stoppingToken)
{
    var tasks = _channelManager.DeviceNames
        .Select(name => ConsumeDeviceAsync(name, stoppingToken))
        .ToArray();

    _logger.LogInformation("Channel consumers started for {Count} devices", tasks.Length);

    await Task.WhenAll(tasks);

    _logger.LogInformation("All channel consumers completed");
}
```

**ConsumeDeviceAsync(string deviceName, CancellationToken ct):**
```csharp
private async Task ConsumeDeviceAsync(string deviceName, CancellationToken ct)
{
    var reader = _channelManager.GetReader(deviceName);

    await foreach (var envelope in reader.ReadAllAsync(ct))
    {
        try
        {
            var msg = new SnmpOidReceived
            {
                Oid        = envelope.Oid,
                AgentIp    = envelope.AgentIp,
                DeviceName = envelope.DeviceName,   // pre-resolved by listener
                Value      = envelope.Value,
                Source     = SnmpSource.Trap,        // ALWAYS Trap for trap-originated events
                TypeCode   = envelope.TypeCode,
            };

            _pipelineMetrics.IncrementTrapReceived();    // snmp.trap.received (PMET-06)
            await _sender.Send(msg, ct);                 // ISender.Send — behaviors execute
        }
        catch (OperationCanceledException) when (ct.IsCancellationRequested)
        {
            break;     // graceful shutdown — exit consumer loop
        }
        catch (Exception ex)
        {
            _logger.LogWarning(ex,
                "Error processing varbind {Oid} for {DeviceName}",
                envelope.Oid, deviceName);
            // continue to next envelope — do not crash consumer
        }
    }
}
```

**Key design notes:**
- `ReadAllAsync(ct)` completes naturally when:
  (a) `ct` is cancelled (host shutdown), OR
  (b) the channel writer is marked complete via `DeviceChannelManager.CompleteAll()` called from `SnmpTrapListenerService.StopAsync`
- `SnmpOidReceived.DeviceName` is set from `envelope.DeviceName` (pre-resolved at listener time). This means `ValidationBehavior`'s `msg.DeviceName is null` check is bypassed (correct, because it's already resolved). The device name is known; only trap path from unknown devices would have null, and those are dropped by the listener.
- `SnmpOidReceived.Source = SnmpSource.Trap` — all trap-originated events get `source=trap` label in OTel metrics
- `IncrementTrapReceived()` is called BEFORE `ISender.Send()` — it counts varbinds that entered the pipeline, regardless of pipeline outcome (consistent with SC#1 which says snmp.trap.received reflects received-not-dropped)
