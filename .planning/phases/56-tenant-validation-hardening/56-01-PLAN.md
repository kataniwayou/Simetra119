  **Finding #1 — IP resolution warning (after line ~250):**
  After `metric.Ip = resolvedIp;` but before `cleanMetrics.Add(metric);`, add:
  ```csharp
  if (resolvedIp == metric.Ip && !System.Net.IPAddress.TryParse(metric.Ip, out _))
  {
      logger.LogWarning(
          "Tenant '{TenantId}' Metrics[{Index}] IP '{Ip}' was not resolved -- possible routing mismatch",
          tenantId, j, metric.Ip);
  }
  ```
  IMPORTANT: This check must come AFTER `metric.Ip = resolvedIp;` because at that point resolvedIp == metric.Ip only if the loop didn't find a match. The condition also requires the original value to NOT be a valid IP (hostname that failed resolution). Warn-only — metric is NOT skipped.

  NOTE: The `resolvedIp == metric.Ip` comparison works because `metric.Ip` was just set to `resolvedIp` on the preceding line. If the AllDevices loop found a match, `resolvedIp` would differ from the original `metric.Ip`. But since assignment happened first, we need to compare the assigned value with itself — which is always true. The actual check here should compare the ORIGINAL metric.Ip value before assignment. Fix: capture `var originalIp = metric.Ip;` BEFORE the AllDevices loop, then after assignment check `if (resolvedIp == originalIp && !System.Net.IPAddress.TryParse(originalIp, out _))`. Actually, looking more carefully: `resolvedIp` is initialized to `metric.Ip` before the loop, and only changed if a match is found. Then `metric.Ip = resolvedIp` is set. So after that line, `metric.Ip` IS `resolvedIp`. The check `resolvedIp == metric.Ip` is trivially always true. The RESEARCH.md example places the check AFTER `metric.Ip = resolvedIp`, but uses the variable `resolvedIp` which was set before the assignment. The correct approach: place the check BEFORE `metric.Ip = resolvedIp;`:
  ```csharp
  if (resolvedIp == metric.Ip && !System.Net.IPAddress.TryParse(metric.Ip, out _))
  {
      logger.LogWarning(
          "Tenant '{TenantId}' Metrics[{Index}] IP '{Ip}' was not resolved -- possible routing mismatch",
          tenantId, j, metric.Ip);
  }
  metric.Ip = resolvedIp;
  ```
  At this point `resolvedIp == metric.Ip` is true only if the AllDevices loop found no match (resolvedIp was initialized to metric.Ip and never changed). This is the correct placement.
