```csharp
case SnmpType.Counter32:
{
    var currentValue = ((Counter32)notification.Value).ToUInt32();
    var didEmit = _deltaEngine.RecordDelta(
        notification.Oid,
        agent,
        source,
        metricName,
        SnmpType.Counter32,
        currentValue,
        notification.SysUpTimeCentiseconds);
    if (didEmit) _pipelineMetrics.IncrementHandled();
    break;
}

case SnmpType.Counter64:
{
    var currentValue = ((Counter64)notification.Value).ToUInt64();
    var didEmit = _deltaEngine.RecordDelta(
        notification.Oid,
        agent,
        source,
        metricName,
        SnmpType.Counter64,
        currentValue,
        notification.SysUpTimeCentiseconds);
    if (didEmit) _pipelineMetrics.IncrementHandled();
    break;
}
```

**CRITICAL:** `Counter32.ToUInt32()` returns `uint` which implicitly converts to `ulong` for the RecordDelta parameter. `Counter64.ToUInt64()` returns `ulong` directly. IncrementHandled is ONLY called when RecordDelta returns true (delta was emitted). On first-poll baseline (returns false), IncrementHandled is NOT called -- consistent with the Phase 3 pattern where unrecorded metrics do not increment the handled counter.

Update the class XML doc comment to remove "Counter32 and Counter64 are intentionally deferred to Phase 4" and replace with a note that counters flow through `ICounterDeltaEngine`.

    Assert.Equal("poll", record.Source);
