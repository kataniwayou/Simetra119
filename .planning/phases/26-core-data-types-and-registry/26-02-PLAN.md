    **ITenantVectorRegistry.cs** — `SnmpCollector.Pipeline` namespace:
    - `public interface ITenantVectorRegistry`
    - `IReadOnlyList<PriorityGroup> Groups { get; }` — sorted priority groups for evaluation
    - `int TenantCount { get; }` — computed at build time
    - `int SlotCount { get; }` — computed at build time
    - `bool TryRoute(string ip, int port, string metricName, out IReadOnlyList<MetricSlotHolder> holders)` — routing index lookup
    - Follow `IOidMapService` pattern for XML docs

    **TenantVectorRegistry.cs** — `SnmpCollector.Pipeline` namespace:
    - `public sealed class TenantVectorRegistry : ITenantVectorRegistry`
    - Constructor takes `ILogger<TenantVectorRegistry>`
    - Two volatile fields:
      - `private volatile IReadOnlyList<PriorityGroup> _groups = Array.Empty<PriorityGroup>();`
      - `private volatile FrozenDictionary<RoutingKey, IReadOnlyList<MetricSlotHolder>> _routingIndex = FrozenDictionary<RoutingKey, IReadOnlyList<MetricSlotHolder>>.Empty;`
    - `public IReadOnlyList<PriorityGroup> Groups => _groups;`
    - `public int TenantCount { get; private set; }` and `public int SlotCount { get; private set; }` — set during Reload before volatile swap
    - `public bool TryRoute(...)` — delegates to `_routingIndex.TryGetValue(new RoutingKey(ip, port, metricName), out holders!)`
    - `public void Reload(TenantVectorOptions options)` — follows RESEARCH skeleton exactly:
      1. Capture `_groups` as oldGroups
      2. Build old slot lookup: `Dictionary<(string TenantId, string Ip, int Port, string MetricName), MetricSlotHolder>` using `StringTupleComparer.Instance` for case-insensitive matching
      3. Iterate `options.Tenants`, build MetricSlotHolders per metric, carry over old values via ReadSlot()/WriteValue()
      4. Group tenants by priority using SortedDictionary (ascending key order = highest priority first)
      5. Build routing index: `Dictionary<RoutingKey, List<MetricSlotHolder>>` with RoutingKeyComparer.Instance, then `.ToFrozenDictionary(RoutingKeyComparer.Instance)`
      6. Set TenantCount and SlotCount before swap
      7. Volatile swap: `_groups = newGroups; _routingIndex = newRoutingIndex;`
      8. Log diff: tenants added/removed/unchanged, slots carried over vs total

    - `StringTupleComparer` — internal sealed class (can be nested in TenantVectorRegistry or separate in same file):
      - Implements `IEqualityComparer<(string TenantId, string Ip, int Port, string MetricName)>`
      - OrdinalIgnoreCase for all string fields
      - `HashCode.Combine` with OrdinalIgnoreCase.GetHashCode for strings

    - **IMPORTANT anti-patterns:**
      - Do NOT use `lock` anywhere
      - Do NOT use `Interlocked.Exchange` — Volatile.Write via volatile keyword is sufficient
      - Do NOT put routing/evaluation logic in the registry
      - Do NOT use `ConcurrentDictionary` — full rebuild + volatile swap is the pattern
      - Value carry-over MUST use ReadSlot()/WriteValue() — never access _slot directly
