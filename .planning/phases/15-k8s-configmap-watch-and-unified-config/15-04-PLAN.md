<objective>
Update K8s manifests: RBAC for ConfigMap watch permissions and ConfigMap YAML to use a single
unified simetra-config.json key replacing the legacy oidmap-*.json and devices.json keys.

Purpose: Without RBAC, the ConfigMap watcher fails with 403 Forbidden. Without ConfigMap YAML
updates, the unified key doesn't exist in K8s.

OID-to-metric-name mapping service. The oidmap-obp.json and oidmap-npb.json keys were SnmpCollector-era
artifacts that the Simetra project never consumed.

    **RBAC updates (both `deploy/k8s/rbac.yaml` and `deploy/k8s/production/rbac.yaml`):**

    1. Rename the Role from `simetra-lease-role` to `simetra-role` (now grants more than just lease access).
    2. Add a new rule for configmaps:
       ```yaml
       - apiGroups: [""]
         resources: ["configmaps"]
         verbs: ["get", "list", "watch"]
       ```
    3. Update the RoleBinding `roleRef.name` to match the renamed Role: `simetra-role`.
    4. Rename the RoleBinding from `simetra-lease-binding` to `simetra-binding`.
    5. Keep the existing lease rule unchanged.

    Final RBAC structure (both files):
    ```yaml
    apiVersion: rbac.authorization.k8s.io/v1
    kind: Role
    metadata:
      name: simetra-role
      namespace: simetra
    rules:
    - apiGroups: ["coordination.k8s.io"]
      resources: ["leases"]
      verbs: ["get", "list", "watch", "create", "update", "patch", "delete"]
    - apiGroups: [""]
      resources: ["configmaps"]
      verbs: ["get", "list", "watch"]
    ---
    apiVersion: rbac.authorization.k8s.io/v1
    kind: RoleBinding
    metadata:
      name: simetra-binding
      namespace: simetra
    roleRef:
      apiGroup: rbac.authorization.k8s.io
      kind: Role
      name: simetra-role
    subjects:
    - kind: ServiceAccount
      name: simetra-sa
      namespace: simetra
    ```
