    **DeviceRegistry.cs** -- Make mutable with atomic swap (matching OidMapService FrozenDictionary pattern from SnmpCollector):
    1. Change `_devices` and `_devicesByName` from `Dictionary` to `volatile FrozenDictionary` fields.
       Add `using System.Collections.Frozen;`.
    2. Store `IEnumerable<IDeviceModule> _modules` as a private readonly field (needed for Reload).
    3. In constructor: build FrozenDictionaries using the same logic, store modules reference.
    4. Add public method `void Reload(List<DeviceOptions> devices)`:
       - Accepts new device list (from ConfigMap watcher).
       - Uses `Dns.GetHostAddressesAsync()` is NOT needed here -- DeviceRegistry uses `IPAddress.Parse(d.IpAddress)`
         which works for both IP addresses and DNS names that were already resolved. However, K8s Service DNS names
         like "obp-simulator.simetra.svc.cluster.local" need DNS resolution. So add an async variant:
         `async Task ReloadAsync(List<DeviceOptions> devices)` that uses `Dns.GetHostAddressesAsync` for DNS names.
       - Actually, looking at the current constructor, it calls `IPAddress.Parse(d.IpAddress)` which will FAIL
         for DNS names. The Phase 14 work must have resolved DNS to IPs at startup. Check the constructor --
         it does `IPAddress.Parse(d.IpAddress).MapToIPv4()`. DNS names are resolved BEFORE being stored in config.
         For reload, we need to handle DNS resolution. Add `async Task ReloadAsync(List<DeviceOptions> devices)`:
         - For each device, attempt `IPAddress.TryParse(d.IpAddress, out var ip)`. If that fails (DNS name),
           use `await Dns.GetHostAddressesAsync(d.IpAddress)` and take the first IPv4 result.
         - Rebuild both FrozenDictionary maps and atomically swap them (volatile write).
         - Re-apply virtual device module auto-registration (same logic as constructor).
         - Log the diff: devices added, removed, changed.
    5. Update `TryGetDevice` and `TryGetDeviceByName` to read from the volatile fields (they already do
       dictionary lookup, which is safe with volatile FrozenDictionary swap).

    **IDeviceRegistry.cs** -- Add:

      // Maps SNMP OID strings to human-readable metric names.

        // System health (4 OIDs)
        // ... all 68 NPB OIDs with inline comments

      // --- Devices ---
      // Each device defines its SNMP connection details and metric poll groups.

          "Name": "OBP-01",
          "IpAddress": "obp-simulator.simetra.svc.cluster.local",
          // ... complete OBP-01 config with MetricPolls
        },
        {
          // NPB-01 device entry (same as current devices.json)
          "Name": "NPB-01",
          "IpAddress": "npb-simulator.simetra.svc.cluster.local",
          // ... complete NPB-01 config with MetricPolls

      ]
