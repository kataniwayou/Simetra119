<objective>
Create the unified config data model, JSONC parser, and make DeviceRegistry/PollDefinitionRegistry/
JobIntervalRegistry/LivenessVectorService mutable for runtime config reload. These are the foundational
types that Plans 02 and 03 depend on.

Purpose: Enable runtime config reload by establishing the data model for unified config and making
registries support atomic swap. Without these, the ConfigMapWatcherService and DynamicPollScheduler
cannot function.

Output: SimetraConfigModel POCO (Devices-only, no OidMap), JsoncParser helper, mutable DeviceRegistry
with ReloadAsync, PollDefinitionRegistry with Reload, JobIntervalRegistry with Unregister,
LivenessVectorService with Remove, and a local dev simetra-config.json file.

IMPORTANT CODEBASE CONTEXT:
- The Simetra project has NO OidMapService. OID resolution is fully embedded in MetricPollOptions.Oids[]
  (each poll definition carries its OID list with PropertyName, Role, EnumMap).
- The configmap oidmap-*.json keys (OidMap dictionaries mapping OID->metric-name) are SnmpCollector-era
  artifacts. The Simetra project does NOT use them.
- Therefore SimetraConfigModel should NOT have an OidMap property. The unified config is just a Devices
  wrapper (same structure as the existing DevicesOptions but parsed from JSONC).
- PollDefinitionRegistry is built once at startup from IOptions<DevicesOptions>. When devices change at
  runtime, MetricPollJob calls _pollRegistry.TryGetDefinition() to find its poll definition. New devices
  added dynamically won't have their definitions in the registry, so new jobs will fail. A Reload method
  is needed.
</objective>

  <action>
    **SimetraConfigModel.cs** in `Simetra.Configuration` namespace:
    - Sealed class with ONE property:
      - `List<DeviceOptions> Devices { get; set; } = [];`
    - NO OidMap property. The Simetra project does NOT have an OidMapService.
      OID resolution is fully embedded in MetricPollOptions.Oids[] (each poll carries its OID list
      with PropertyName, Role, EnumMap). There is no separate OID-to-metric-name mapping.
    - Add XML doc comments explaining this represents the unified simetra-config.json structure
      used by ConfigMap watcher and local dev fallback.

    **JsoncParser.cs** in `Simetra.Configuration` namespace:
    - Static sealed class with a static readonly `JsonSerializerOptions` field named `Options`:
      - `ReadCommentHandling = JsonCommentHandling.Skip`
      - `AllowTrailingCommas = true`
      - `PropertyNameCaseInsensitive = true`
    - Static method `SimetraConfigModel Parse(string jsonContent)` that:
      - Calls `JsonSerializer.Deserialize<SimetraConfigModel>(jsonContent, Options)`
      - Throws `InvalidOperationException` with clear message if result is null
    - Add `using System.Text.Json;` as needed.
    - Add XML doc comments.

    **DeviceRegistry.cs** -- Make mutable with atomic swap:
    1. Change `_devices` and `_devicesByName` from `Dictionary` to `volatile FrozenDictionary` fields.
       Add `using System.Collections.Frozen;`.
    2. Store `IEnumerable<IDeviceModule> _modules` as a private readonly field (needed for Reload).
    3. In constructor: build FrozenDictionaries using the same logic, store modules reference.
    4. Add public method `async Task ReloadAsync(List<DeviceOptions> devices)`:
       - For each device, attempt `IPAddress.TryParse(d.IpAddress, out var ip)`. If that fails (DNS name),
         use `await Dns.GetHostAddressesAsync(d.IpAddress)` and take the first IPv4 result.
       - Rebuild both FrozenDictionary maps and atomically swap them (volatile write).
       - Re-apply virtual device module auto-registration (same logic as constructor).
       - Log the diff: devices added, removed, changed (inject ILogger<DeviceRegistry>).

    **IDeviceRegistry.cs** -- Add:
    - `Task ReloadAsync(List<DeviceOptions> devices);` with XML doc comment explaining it rebuilds the
      registry from new device config, resolving DNS names asynchronously.

    **PollDefinitionRegistry.cs** -- Add mutable Reload method:
    1. Store `IEnumerable<IDeviceModule> _modules` and `ILogger<PollDefinitionRegistry> _logger` as
       private readonly fields (already available from constructor).
    2. Add public method `void Reload(List<DeviceOptions> devices)`:
       - Clear `_definitions`, `_statePollDefinitions`, `_metricPollDefinitions`.
       - Re-run the exact same indexing logic as the constructor but using the new `devices` list
         instead of `devicesOptions.Value.Devices`.
       - Index modules by DeviceType, apply state poll definitions, then metric poll definitions
         with the same collision-check and duplicate-skip logic.
       - Before re-indexing, set `poll.Source = MetricPollSource.Configuration` on each MetricPoll
         (same as PostConfigure does -- needed because the JSONC-parsed config won't have Source set).
    3. This is a synchronous method because PollDefinitionRegistry uses no async operations.
       It must be called AFTER DeviceRegistry.ReloadAsync completes.

    **IPollDefinitionRegistry.cs** -- Add:
    - `void Reload(List<DeviceOptions> devices);` with XML doc comment explaining it rebuilds all
      poll definitions from new device config. Called during config reload after DeviceRegistry
      is updated. New metric-poll jobs will fail TryGetDefinition() until this is called.

    **IJobIntervalRegistry.cs** -- Add:
    - `void Unregister(string jobKey);` with XML doc comment explaining it removes a job's interval
      registration, called when removing Quartz jobs during device config reload.

    **JobIntervalRegistry.cs** -- Add:
    - `public void Unregister(string jobKey) { _intervals.Remove(jobKey); }` implementation.

    **ILivenessVectorService.cs** -- Add:
    - `void Remove(string jobKey);` with XML doc comment explaining it removes a stale liveness stamp
      for a job that no longer exists, preventing false-positive staleness alerts.

    **LivenessVectorService.cs** -- Add:
    - `public void Remove(string jobKey) { _stamps.TryRemove(jobKey, out _); }` implementation.
  </action>
