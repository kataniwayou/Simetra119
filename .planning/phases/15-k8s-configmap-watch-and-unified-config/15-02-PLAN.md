<objective>
Create the ConfigMapWatcherService (K8s API watch with reconnect loop) and DynamicPollScheduler
(Quartz job reconciliation). These are the two runtime services that enable live config reload.

Purpose: ConfigMapWatcherService replaces file-based IOptionsMonitor hot-reload with direct K8s API
watch for instant config change detection. DynamicPollScheduler enables adding/removing poll jobs
at runtime without pod restart.

Output: ConfigMapWatcherService.cs, DynamicPollScheduler.cs, IDynamicPollScheduler.cs

CRITICAL WIRING: ConfigMapWatcherService.ApplyConfigAsync must call THREE things in order:
1. DeviceRegistry.ReloadAsync (update device lookup maps)
2. PollDefinitionRegistry.Reload (rebuild poll definitions so TryGetDefinition works for new devices)
3. DynamicPollScheduler.ReconcileJobsAsync (add/remove Quartz jobs)

Without step 2, new metric-poll jobs created by step 3 will fail because MetricPollJob.Execute calls
_pollRegistry.TryGetDefinition() and gets false for new devices.
</objective>

  <name>Task 1: Create ConfigMapWatcherService</name>
  <files>

  <action>
    Create `ConfigMapWatcherService.cs` in `Simetra.Services` namespace. Follow the `K8sLeaseElection`
    BackgroundService pattern already in the codebase.

    **Class design:**
    - `public sealed class ConfigMapWatcherService : BackgroundService`
    - Constructor injects:
      - `IKubernetes kubeClient` (already registered as singleton in DI)
      - `IDeviceRegistry deviceRegistry`
      - `IPollDefinitionRegistry pollRegistry`
      - `IDynamicPollScheduler pollScheduler`
      - `ILogger<ConfigMapWatcherService> logger`
    - Private fields:
      - `private readonly string _namespace` -- read from `NAMESPACE` env var, fallback to `"simetra"`
      - `private const string ConfigMapName = "simetra-config";`
      - `private const string ConfigKey = "simetra-config.json";`
      - `private readonly SemaphoreSlim _reloadLock = new(1, 1);` -- serializes concurrent reload events

    **ExecuteAsync(CancellationToken stoppingToken):**
    1. Initial load: call `await LoadAndApplyConfigAsync(stoppingToken)` to load current ConfigMap state.
    2. Enter reconnect loop: `while (!stoppingToken.IsCancellationRequested)`
    3. Inside loop, try-catch:
       - Call `_kubeClient.CoreV1.ListNamespacedConfigMapWithHttpMessagesAsync(_namespace, fieldSelector: $"metadata.name={ConfigMapName}", watch: true, cancellationToken: stoppingToken)`
       - Use `await foreach (var (type, configMap) in response.WatchAsync<V1ConfigMap, V1ConfigMapList>(cancellationToken: stoppingToken))`
       - On `WatchEventType.Modified`: call `await HandleConfigMapModifiedAsync(configMap, stoppingToken)`
    4. Catch `OperationCanceledException` when `stoppingToken.IsCancellationRequested` -> break
    5. Catch `Exception` -> log warning "ConfigMap watch disconnected, reconnecting in 5s",
       `await Task.Delay(TimeSpan.FromSeconds(5), stoppingToken)`

    **LoadAndApplyConfigAsync(CancellationToken ct):**
    1. Read ConfigMap: `var cm = await _kubeClient.CoreV1.ReadNamespacedConfigMapAsync(ConfigMapName, _namespace, cancellationToken: ct)`
    2. Extract key: `cm.Data.TryGetValue(ConfigKey, out var jsonContent)` -- log error and return if missing
    3. Parse: `var config = JsoncParser.Parse(jsonContent)`
    4. Apply: call `await ApplyConfigAsync(config, ct)`
    5. Log: "ConfigMap initial load complete: {DeviceCount} devices"

    **HandleConfigMapModifiedAsync(V1ConfigMap configMap, CancellationToken ct):**
    1. Extract key content -- return early if key missing
    2. Parse with `JsoncParser.Parse` -- wrap in try-catch, log error on parse failure, do NOT apply bad config
    3. Call `await ApplyConfigAsync(config, ct)`

    **ApplyConfigAsync(SimetraConfigModel config, CancellationToken ct):**
    1. Acquire `_reloadLock`: `await _reloadLock.WaitAsync(ct)` in try-finally
    2. Set Source on all metric polls: foreach device.MetricPolls, set poll.Source = MetricPollSource.Configuration
    3. Call `await _deviceRegistry.ReloadAsync(config.Devices)` -- update device lookup maps
    4. Call `_pollRegistry.Reload(config.Devices)` -- rebuild poll definitions (CRITICAL: must happen
       BEFORE ReconcileJobsAsync so new jobs can find their definitions via TryGetDefinition)
    5. Call `await _pollScheduler.ReconcileJobsAsync(config.Devices, ct)` -- add/remove Quartz jobs
    6. Log: "Config reload applied: {DeviceCount} devices"

    **Required usings:**
    - `using k8s;`
    - `using k8s.Models;`
    - `using k8s.Autorest;` (for WatchAsync extension)
    - `using Simetra.Configuration;`
    - `using Simetra.Pipeline;`

    **Important implementation notes:**
    - The watch API returns a `Task<HttpOperationResponse<V1ConfigMapList>>`. You need to call
      `.WatchAsync<V1ConfigMap, V1ConfigMapList>()` on the response to get the async enumerable.
    - Match the error handling pattern from K8sLeaseElection: OperationCanceledException propagates
      for graceful shutdown, all other exceptions are logged and retried.
    - Add XML doc comments on the class and all public/protected methods.

    **Create `IDynamicPollScheduler.cs`** in `Simetra.Services` namespace:
    - Interface with single method: `Task ReconcileJobsAsync(List<DeviceOptions> newDevices, CancellationToken ct)`
    - XML doc comment explaining it reconciles Quartz metric-poll jobs with the new device config,
      adding new jobs and removing stale ones.

    **Create `DynamicPollScheduler.cs`** in `Simetra.Services` namespace:
    - `public sealed class DynamicPollScheduler : IDynamicPollScheduler`
    - Constructor injects:
      - `ISchedulerFactory schedulerFactory`
      - `IJobIntervalRegistry intervalRegistry`
      - `ILivenessVectorService livenessVector`
      - `IEnumerable<IDeviceModule> modules`
      - `ILogger<DynamicPollScheduler> logger`
    - Store modules in a `Dictionary<string, IDeviceModule>` keyed by DeviceType (same pattern as
      ServiceCollectionExtensions.AddScheduling).

    **ReconcileJobsAsync(List<DeviceOptions> newDevices, CancellationToken ct):**

    1. Get the Quartz scheduler: `var scheduler = await _schedulerFactory.GetScheduler(ct)`

    2. Build DESIRED state -- a `Dictionary<string, (JobKey Key, DeviceOptions Device, MetricPollOptions Poll)>`
       keyed by job key name string. For each device in newDevices:
       - For each MetricPoll in device.MetricPolls:
         - Skip if a module already defines a state poll with this PollKey for this device type
           (same collision check as AddScheduling)
         - Job key name: `"metric-poll-{device.Name}-{poll.MetricName}"`
         - Add to desired set

    3. Get CURRENT state -- query Quartz for existing metric-poll jobs:
       - `var allJobKeys = await scheduler.GetJobKeys(GroupMatcher<JobKey>.AnyGroup(), ct)`
       - Filter to those whose Name starts with `"metric-poll-"`
       - Collect into a `HashSet<string>` of job key names

    4. **Remove stale jobs** (in current but not in desired):
       - For each stale key: `await scheduler.DeleteJob(new JobKey(key), ct)`
       - Call `_intervalRegistry.Unregister(key)`
       - Call `_livenessVector.Remove(key)`
       - Log: "Removed poll job: {JobKey}"

    5. **Add new jobs** (in desired but not in current):
       - For each new entry:
         - Set `poll.Source = MetricPollSource.Configuration` (same as PostConfigure in AddSimetraConfiguration)
         - Create JobBuilder: `JobBuilder.Create<MetricPollJob>().WithIdentity(jobKey)
           .UsingJobData("deviceName", device.Name)
           .UsingJobData("pollKey", poll.MetricName)
           .UsingJobData("intervalSeconds", poll.IntervalSeconds).Build()`
         - Create TriggerBuilder with same pattern as AddScheduling:
           `TriggerBuilder.Create().WithIdentity($"{keyName}-trigger").StartNow()
           .WithSimpleSchedule(s => s.WithIntervalInSeconds(poll.IntervalSeconds)
           .RepeatForever().WithMisfireHandlingInstructionNextWithRemainingCount()).Build()`
         - `await scheduler.ScheduleJob(job, trigger, ct)`
         - `_intervalRegistry.Register(keyName, poll.IntervalSeconds)`
         - Log: "Added poll job: {JobKey} (interval: {Interval}s)"

    6. **Update existing jobs** (in both current and desired -- check if interval changed):
       - For each key in both sets:
         - Get current triggers: `var triggers = await scheduler.GetTriggersOfJob(new JobKey(key), ct)`
         - Compare interval from trigger to desired interval
         - If different: reschedule trigger with `await scheduler.RescheduleJob(trigger.Key, newTrigger, ct)`
         - Update `_intervalRegistry.Register(key, newInterval)`
         - Log: "Rescheduled poll job: {JobKey} ({OldInterval}s -> {NewInterval}s)"

    7. Log summary: "Job reconciliation complete: +{Added} added, -{Removed} removed, ~{Updated} updated"

    **Important notes:**
    - State poll jobs (from IDeviceModule.StatePollDefinitions) are NOT managed by DynamicPollScheduler.
      They are static and registered at startup. Only metric-poll jobs (Source=Configuration) are dynamic.
    - Use `GroupMatcher<JobKey>.AnyGroup()` instead of `GroupEquals("DEFAULT")` since Quartz may use
      different group names.
    - Match the MetricPollJob JobData keys exactly: "deviceName", "pollKey", "intervalSeconds" -- these
      are read by MetricPollJob at execution time.
    - Add XML doc comments on the class and all public methods.

    **Required usings:**
    - `using Quartz;`
    - `using Quartz.Impl.Matchers;`
    - `using Simetra.Configuration;`
    - `using Simetra.Devices;`
    - `using Simetra.Jobs;`
    - `using Simetra.Pipeline;`
