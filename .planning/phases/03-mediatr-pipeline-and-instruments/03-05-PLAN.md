```csharp
/// <summary>
/// Registers the MediatR pipeline with behaviors in execution order, TaskWhenAllPublisher
/// for parallel handler dispatch, and all Phase 3 singleton services (PipelineMetricService,
/// SnmpMetricFactory).
/// <para>
/// Behavior registration order (first registered = outermost = runs first pre-handler):
/// 1. LoggingBehavior — logs every event at Debug
/// 2. ExceptionBehavior — catches all downstream failures
/// 3. ValidationBehavior — rejects malformed OID or unknown device
/// 4. OidResolutionBehavior — resolves OID to metric_name
/// </para>
/// <para>
/// Must be called AFTER AddSnmpConfiguration (behaviors depend on IDeviceRegistry,
/// IOidMapService, and IOptions&lt;SiteOptions&gt;).
/// </para>
/// </summary>
public static IServiceCollection AddSnmpPipeline(this IServiceCollection services)
{
    // --- MediatR registration ---
    // AddOpenBehavior order: first registered = outermost (PIPE-08).
    // MediatR reverses the IEnumerable<IPipelineBehavior> before chaining,
    // so first-registered wraps all subsequent behaviors.
    services.AddMediatR(cfg =>
    {
        cfg.RegisterServicesFromAssemblyContaining<SnmpOidReceived>();
        cfg.NotificationPublisher = new TaskWhenAllPublisher();  // PIPE-09: singleton instance
        cfg.AddOpenBehavior(typeof(LoggingBehavior<,>));         // 1st = outermost
        cfg.AddOpenBehavior(typeof(ExceptionBehavior<,>));       // 2nd
        cfg.AddOpenBehavior(typeof(ValidationBehavior<,>));      // 3rd
        cfg.AddOpenBehavior(typeof(OidResolutionBehavior<,>));   // 4th = innermost
    });

    // --- Phase 3 singletons ---
    services.AddSingleton<PipelineMetricService>();
    services.AddSingleton<ISnmpMetricFactory, SnmpMetricFactory>();

    return services;
}
```

**CRITICAL registration details:**
1. `cfg.NotificationPublisher = new TaskWhenAllPublisher()` -- sets the INSTANCE directly, making it a singleton. Do NOT use `cfg.NotificationPublisherType = typeof(TaskWhenAllPublisher)` which would use `cfg.ServiceLifetime` (default Transient).
2. Behavior order MUST be: Logging, Exception, Validation, OidResolution. This is the PIPE-08 requirement.
3. `RegisterServicesFromAssemblyContaining<SnmpOidReceived>()` auto-discovers `OtelMetricHandler` (INotificationHandler<SnmpOidReceived>) in the same assembly.
4. PipelineMetricService is a concrete singleton (no interface) -- behaviors inject it directly.
5. SnmpMetricFactory is registered via its interface for testability.
