public void RecordGauge(string metricName, string oid, string agent, string source, double value)

**RecordInfo** implementation -- records gauge value 1.0 with a 6th `value` label:
```csharp
public void RecordInfo(string metricName, string oid, string agent, string source, string value)
{
    var truncated = value.Length > 128
        ? string.Concat(value.AsSpan(0, 125), "...")
        : value;

    var gauge = GetOrCreateGauge("snmp_info");
    gauge.Record(1.0, new TagList
    {
        { "site_name",   _siteName },
        { "metric_name", metricName },
        { "oid",         oid },
        { "agent",       agent },
        { "source",      source },
        { "value",       truncated }
    });
}
```

**CRITICAL:** The `value` label on snmp_info is truncated at 128 characters with `"..."` suffix (CONTEXT.md locked decision for cardinality protection). Use `string.Concat(value.AsSpan(0, 125), "...")` for efficient truncation (125 + 3 = 128).

The `snmp_counter` instrument will be pre-created but not yet publicly exposed. Phase 4 will add a `RecordCounter` method.

**Handle method** -- dispatches based on `notification.TypeCode`:

```csharp
public Task Handle(SnmpOidReceived notification, CancellationToken cancellationToken)
{
    var metricName = notification.MetricName ?? OidMapService.Unknown;
    var agent = notification.DeviceName ?? notification.AgentIp.ToString();
    var source = notification.Source.ToString().ToLowerInvariant();

    switch (notification.TypeCode)
    {
        case SnmpType.Integer32:
        case SnmpType.Gauge32:
        case SnmpType.TimeTicks:
            RecordGauge(notification, metricName, agent, source);
            _pipelineMetrics.IncrementHandled();
            break;

        case SnmpType.Counter32:
        case SnmpType.Counter64:
            // Phase 3: do NOT record -- delta engine (Phase 4) must exist first
            _logger.LogDebug(
                "Counter OID {Oid} TypeCode={TypeCode} deferred to Phase 4 delta engine",
                notification.Oid, notification.TypeCode);
            break;

        case SnmpType.OctetString:
        case SnmpType.IPAddress:
        case SnmpType.ObjectIdentifier:
            RecordInfo(notification, metricName, agent, source);
            _pipelineMetrics.IncrementHandled();
            break;

        default:
            _logger.LogWarning(
                "Unrecognized SnmpType {TypeCode} for OID {Oid}, dropping",
                notification.TypeCode, notification.Oid);
            break;
    }

    return Task.CompletedTask;
}
```

**RecordGauge helper** (private):
```csharp
private void RecordGauge(SnmpOidReceived n, string metricName, string agent, string source)
{
    // ISnmpData.ToInt32() works for Integer32, Gauge32, TimeTicks
    var value = Convert.ToDouble(n.Value.ToInt32());
    _metricFactory.RecordGauge(metricName, n.Oid, agent, source, value);
}
```

**RecordInfo helper** (private):
```csharp
private void RecordInfo(SnmpOidReceived n, string metricName, string agent, string source)
{
    var value = n.Value.ToString() ?? string.Empty;
    _metricFactory.RecordInfo(metricName, n.Oid, agent, source, value);
}
```

**CRITICAL dispatch decisions (from CONTEXT.md):**
- TimeTicks -> snmp_gauge (operators want current uptime value, not deltas)
- Counter32/Counter64 -> deferred to Phase 4 (logged at Debug, NOT recorded, NOT counted as handled)
- Unrecognized types (Opaque, NoSuchObject, EndOfMibView, NoSuchInstance) -> Warning log, dropped, NOT counted as handled
- IncrementHandled() only on successful gauge or info recording

**Note on ToInt32():** SharpSnmpLib's `Integer32.ToInt32()`, `Gauge32.ToInt32()`, and `TimeTicks.ToInt32()` all return the underlying integer value. `TimeTicks.ToInt32()` returns the raw tick count (hundredths of a second). Convert to double for OTel Gauge<double>.

Required usings: `Lextm.SharpSnmpLib`, `MediatR`, `Microsoft.Extensions.Logging`, `SnmpCollector.Telemetry`.
