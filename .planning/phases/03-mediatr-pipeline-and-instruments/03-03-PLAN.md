Open generic `IPipelineBehavior<TNotification, TResponse> where TNotification : INotification`.

**Static compiled regex:**
```csharp
private static readonly Regex OidPattern =
    new(@"^\d+(\.\d+){1,}$", RegexOptions.Compiled);
```
This requires at least 2 arcs (e.g., `1.3`). The `{1,}` ensures one or more dot-digit groups after the first arc.

**Constructor** takes:
- `ILogger<ValidationBehavior<TNotification, TResponse>> logger`
- `PipelineMetricService metrics`
- `IDeviceRegistry deviceRegistry`

**Handle method logic:**
1. `if (notification is not SnmpOidReceived msg) return await next();` -- pass through non-SNMP notifications
2. **OID format check:** `if (!OidPattern.IsMatch(msg.Oid))` -- reject:
   - `_logger.LogWarning("Rejecting SnmpOidReceived: OID={Oid} Agent={Agent} Reason={Reason}", msg.Oid, msg.AgentIp, "InvalidOidFormat")`
   - `_metrics.IncrementRejected()`
   - `return default!` -- short-circuit, do NOT call next()
3. **Unknown device check:** If `msg.DeviceName` is null, try to resolve via registry:
   - `if (msg.DeviceName is null && !_deviceRegistry.TryGetDevice(msg.AgentIp, out var device))`
     - `_logger.LogWarning("Rejecting SnmpOidReceived: OID={Oid} Agent={Agent} Reason={Reason}", msg.Oid, msg.AgentIp, "UnknownDevice")`
     - `_metrics.IncrementRejected()`
     - `return default!`
   - If device IS found and DeviceName was null, set it: `msg.DeviceName = device.Name`
4. All checks passed: `return await next()`

**CONTEXT.md locked decision:** Unknown devices (IP not in registry) are rejected with Warning log -- only configured devices get metrics.

Required usings: `System.Text.RegularExpressions`, `MediatR`, `Microsoft.Extensions.Logging`, `SnmpCollector.Telemetry`.

Open generic `IPipelineBehavior<TNotification, TResponse> where TNotification : INotification`.

**Constructor** takes:
- `IOidMapService oidMapService`

**Handle method logic:**
1. `if (notification is SnmpOidReceived msg)` -- pattern match
2. If matched: `msg.MetricName = _oidMapService.Resolve(msg.Oid)` -- sets MetricName in-place (mutable class pattern)
3. Always: `return await next()` -- OidResolution never short-circuits

This is the simplest behavior. It resolves the OID to a metric_name (or "Unknown" if not in the map) and passes the enriched notification downstream to the handler.

**No logger needed** -- OID resolution is normal flow, not an error condition. Debug-level logging of every resolution would be noise since LoggingBehavior already logs the OID.

Required usings: `MediatR`.
