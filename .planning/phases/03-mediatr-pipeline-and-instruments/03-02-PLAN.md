**LoggingBehavior.cs** -- Open generic `IPipelineBehavior<TNotification, TResponse>`:

```csharp
public sealed class LoggingBehavior<TNotification, TResponse>
    : IPipelineBehavior<TNotification, TResponse>
    where TNotification : INotification
```

Constructor takes `ILogger<LoggingBehavior<TNotification, TResponse>> logger`.

`Handle` method:
1. Check `if (notification is SnmpOidReceived msg)` -- pattern match
2. If matched: `_logger.LogDebug("SnmpOidReceived OID={Oid} Agent={Agent} Source={Source}", msg.Oid, msg.AgentIp, msg.Source)`
3. Always `return await next()` -- never short-circuit, never skip

Required usings: `MediatR`, `Microsoft.Extensions.Logging`.

**ExceptionBehavior.cs** -- Open generic `IPipelineBehavior<TNotification, TResponse>`:

```csharp
public sealed class ExceptionBehavior<TNotification, TResponse>
    : IPipelineBehavior<TNotification, TResponse>
    where TNotification : INotification
```

Constructor takes `ILogger<ExceptionBehavior<TNotification, TResponse>> logger` and `PipelineMetricService metrics`.

`Handle` method:
1. Wrap `return await next()` in a `try` block
2. In `catch (Exception ex)`:
   - Log: `_logger.LogWarning(ex, "Pipeline exception for {NotificationType}", typeof(TNotification).Name)`
   - Increment: `_metrics.IncrementErrors()`
   - Return: `return default!` (which is `Unit.Value` for INotification handlers)
3. **CRITICAL**: The try/catch wraps ONLY the `next()` call, not the whole method body. This ensures only downstream failures are caught (Pitfall 3 from RESEARCH.md).

Required usings: `MediatR`, `Microsoft.Extensions.Logging`, `SnmpCollector.Telemetry`.

**IMPORTANT constraints:**
- Both use `where TNotification : INotification` -- NOT `where TNotification : IRequest<TResponse>` (Pitfall 1 from RESEARCH.md)
- Both are `sealed class` (no inheritance needed)
- Both use the open generic two-parameter form `<TNotification, TResponse>` for `AddOpenBehavior(typeof(Behavior<,>))` compatibility
