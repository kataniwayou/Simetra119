<objective>
Fix host_name resolution to use NODE_NAME env var (K8s Downward API spec.nodeName) instead of HOSTNAME.

Purpose: HOSTNAME in K8s resolves to the pod name (e.g., simetra-7f8b9c6d4-x2k9q) which changes on every restart. The user runs multiple physical servers, each with one K8s cluster and one collector, all pushing to one Prometheus. They need persistent, unique physical server identity. spec.nodeName gives the K8s node hostname (the physical server), which is persistent and unique within a cluster.

Output: All host_name resolution points changed from `HOSTNAME -> MachineName` to `NODE_NAME -> MachineName`. K8s deployment YAMLs inject NODE_NAME via Downward API.
</objective>
