
**Tier 1 — Staleness check (private method, e.g., `HasStaleness`):**
- Input: tenant's holders list
- Filter holders: EXCLUDE holders where `Source == SnmpSource.Trap` OR `IntervalSeconds == 0`
- For each remaining holder:
  - `ReadSlot()` returns null → skip this holder (cannot judge, not stale)
  - Compute age: `DateTimeOffset.UtcNow - slot.Timestamp`
  - Compute grace window: `TimeSpan.FromSeconds(holder.IntervalSeconds * holder.GraceMultiplier)`
  - If `age > graceWindow` → this holder is stale → return true (tenant has staleness)
- If no holder was stale, return false

**Tier 2 — Resolved gate (private method, e.g., `AreAllResolvedViolated`):**
- Input: tenant's holders list
- Filter holders to only `Role == "Resolved"`
- For each Resolved holder:
  - `ReadSlot()` returns null → skip (does not participate in "all violated" check)
  - Check violation using `IsViolated` helper (below)
  - If NOT violated → return false (not all Resolved are violated → gate blocks)
- After loop: if at least one Resolved holder with data was checked and ALL were violated → return true
- If NO Resolved holders have data (all null) → vacuous pass → return true (per CONTEXT.md: empty role groups not possible at runtime, but handle defensively)

**Threshold violation helper (private static, e.g., `IsViolated`):**
- Input: MetricSlotHolder (with non-null ReadSlot already checked by caller)
- `threshold = holder.Threshold`
- If `threshold is null` OR `(threshold.Min is null && threshold.Max is null)` → return true (no threshold = violated)
- `value = slot.Value`
- If `threshold.Min is not null && value < threshold.Min.Value` → return true
- If `threshold.Max is not null && value > threshold.Max.Value` → return true
- Return false (value is in range)
