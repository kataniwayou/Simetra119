**CRITICAL — The tier flow that reaches this code:**
1. Tier 1: Not stale (otherwise would have returned TierResult.Stale already)
2. Tier 2: `AreAllResolvedViolated` returned FALSE (not all Resolved violated → continue)
   - If it had returned TRUE, evaluation would have stopped at ConfirmedBad in 48-02's code
3. NOW we enter Tier 3 — Evaluate check

**Tier 3 — Evaluate metrics check (private method, e.g., `AreAllEvaluateViolated`):**
- Same structure as `AreAllResolvedViolated` but filters to `Role == "Evaluate"`
- For each Evaluate holder:
  - `ReadSlot()` null → skip (does not participate)
  - Check `IsViolated` (reuse the same helper from 48-02)
  - If NOT violated → return false (not all Evaluate violated → no command)
- After loop: if at least one Evaluate holder with data was checked and ALL violated → return true
- If no Evaluate holders have data → vacuous fail (no Evaluate data = no command, return false)

**Tier 3 gate direction:**
- `AreAllEvaluateViolated` returns TRUE → ALL Evaluate violated → proceed to Tier 4 (issue commands)
- `AreAllEvaluateViolated` returns FALSE → NOT all Evaluate violated → tenant is Healthy, return TierResult.Healthy

**Tier 4 — Command dispatch:**
- Iterate `tenant.Commands`
- For each command:
  - Build suppression key: `$"{tenant.Id}:{cmd.Ip}:{cmd.Port}:{cmd.CommandName}"` (include tenant.Id per RESEARCH.md pitfall 2 — isolate tenants)
  - Call `_suppressionCache.TrySuppress(suppressionKey, tenant.SuppressionWindowSeconds)`
  - If suppressed (returns true): `_pipelineMetrics.IncrementCommandSuppressed(tenant.Id)` and continue
  - Build `CommandRequest`: `new CommandRequest(cmd.Ip, cmd.Port, cmd.CommandName, cmd.Value, cmd.ValueType, tenant.Id)` — DeviceName = tenant.Id per locked decision
  - Call `_commandChannel.Writer.TryWrite(request)`
  - If TryWrite returns false: `_logger.LogWarning(...)` + `_pipelineMetrics.IncrementCommandFailed(tenant.Id)`
  - If TryWrite returns true: log at Information level per SNAP-15
