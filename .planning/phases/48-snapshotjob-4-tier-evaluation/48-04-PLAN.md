<task type="auto">
  <name>Task 1: Wire parallel within-group evaluation and advance gate</name>
  <files>src/SnmpCollector/Jobs/SnapshotJob.cs</files>
  <action>
Refactor the Execute method's group loop to support parallel tenant evaluation and advance gate.

**Per-tenant evaluation must return a result** that the advance gate can inspect. Use a result type (TierResult enum or a small record — Claude's discretion) that captures:
- Whether the tenant is stale
- Whether the tenant is commanding (all Evaluate violated, but Resolved NOT all violated)
- Whether the tenant is confirmed-bad (all Resolved violated)
- Whether the tenant is confirmed-healthy (not all Evaluate violated)

**Parallel evaluation within each group:**
- Use `Task.WhenAll` with `Select` over `group.Tenants` per RESEARCH.md Pattern 3
- Pre-allocate a result array with one slot per tenant (indexed by position) per RESEARCH.md recommendation — each task writes to its own index, no locking needed
- Do NOT use `foreach` with sequential `await` — this defeats parallelism

```csharp
var results = new TenantResult[group.Tenants.Count];
await Task.WhenAll(group.Tenants.Select((tenant, index) =>
    EvaluateTenantAsync(tenant, index, results, ct)));
```

**Advance gate after each group:**
- After `await Task.WhenAll(...)`, inspect the results array
- Per CONTEXT.md locked decisions:
  - **Advances when ALL tenants in group are either:**
    - Confirmed-bad (all Resolved violated) — move on
    - Confirmed-healthy (not all Evaluate violated) — no action needed, move on
  - **Blocks when ANY tenant is:**
    - Stale (data missing) — uncertain state, don't cascade
    - Actively commanding (all Evaluate violated, Resolved not all violated) — commands firing, don't cascade
- If advance gate blocks: `break` out of the groups loop — do NOT evaluate lower-priority groups

**Cycle summary logging (SNAP-15):**
After the groups loop (whether completed or broken), log at Debug level:
`"Snapshot cycle complete: {TenantsEvaluated} evaluated, {Commanded} commanded, {Stale} stale"`

Track these counters across all groups. Use `Interlocked.Increment` for counters shared across parallel tasks within a group, OR aggregate from the results arrays after each group's Task.WhenAll completes (simpler, no locking — preferred approach).

**Liveness stamp and correlation ID** remain in the `finally` block (already placed by 48-01). Verify they still execute regardless of advance gate break.
