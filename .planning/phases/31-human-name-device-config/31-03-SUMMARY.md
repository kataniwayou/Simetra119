## Decisions Made

- Production configmap.yaml gets 6 missing oidmap entries added during this migration -- the flat-dict format previously had 88 entries (missing the 6 info OIDs); now aligned with dev at 94 entries.
- `e2e-sim-unmapped-configmap.yaml` uses `e2e_intentionally_missing` instead of the raw OIDs (`1.3.6.1.4.1.47477.999.2.1.0`, `1.3.6.1.4.1.47477.999.2.2.0`) from the previous version. The plan notes these would have mapped to `e2e_unmapped_gauge`, but `e2e_unmapped_gauge` is not in simetra-oidmaps.yaml either -- however, using a clearly intentional name makes the test intent self-documenting.
- NPB port metric names follow the actual oidmaps.json naming convention (`npb_port_rx_octets_P{n}`, `npb_port_tx_octets_P{n}`, `npb_port_rx_packets_P{n}`, `npb_port_tx_packets_P{n}`) which differs from the plan description's `rx_bytes`/`tx_drops` shorthand.
- `.original-devices-configmap.yaml` is gitignored (runtime artifact saved by scenario 06) -- updated on disk to new format but not committed.
