Modify PreferredHeartbeatJob.cs to add the writer path. All changes are in this single file:

**1. New constructor dependencies (add to existing constructor):**
- `IOptions<PodIdentityOptions> podIdentityOptions` — store as `_podIdentity = podIdentityOptions.Value.PodIdentity ?? Environment.MachineName;` (string field, resolved once)
- `IHostApplicationLifetime lifetime` — register callback: `lifetime.ApplicationStarted.Register(() => _isSchedulerReady = true);`
- Add `using Microsoft.Extensions.Hosting;` at the top

**2. New fields:**
- `private readonly string _podIdentity;` — pod identity for holderIdentity on the lease
- `private volatile bool _isSchedulerReady;` — set to true by ApplicationStarted callback
- `private string? _cachedResourceVersion;` — cached from create/replace responses for optimistic concurrency

**3. Restructure Execute method to write-before-read:**
```
Execute(IJobExecutionContext context):
  try {
    // Writer path: only on preferred pod AND only when scheduler is ready
    if (_preferredLeaderService.IsPreferredPod && _isSchedulerReady)
      await WriteHeartbeatLeaseAsync(context.CancellationToken);
    // Silent skip for non-preferred pods — NO log

    // Reader path: all pods, every tick (Phase 85, unchanged)
    await ReadAndUpdateStampFreshnessAsync(context.CancellationToken);
  }
  catch (OperationCanceledException) { throw; }
  catch (Exception ex) { _logger.LogError(ex, "..."); }
  finally { _liveness.Stamp(jobKey); }
```

**4. Add private WriteHeartbeatLeaseAsync method:**
Implements the create-or-replace pattern with cached resourceVersion. Follow the RESEARCH.md Pattern 2 exactly:

- `leaseName = $"{_leaseOptions.Name}-preferred"` (same derivation as reader)
- Build V1LeaseSpec with:
  - `HolderIdentity = _podIdentity`
  - `LeaseDurationSeconds = _leaseOptions.DurationSeconds`
  - `RenewTime = DateTime.UtcNow`
- If `_cachedResourceVersion is null`: try CreateNamespacedLeaseAsync
  - On success: cache resourceVersion from response, return
  - On 409 Conflict: read the existing lease to get resourceVersion, cache it, fall through to replace
  - IMPORTANT: Set `AcquireTime = DateTime.UtcNow` ONLY on the create path
- Replace using cached resourceVersion: ReplaceNamespacedLeaseAsync(body, leaseName, namespace)
  - On success: update cached resourceVersion from response
  - On 409 Conflict or 404: invalidate cache (`_cachedResourceVersion = null`), log Warning
- Catch all non-cancellation exceptions inside WriteHeartbeatLeaseAsync as Warning (consistent with reader path transient error handling). Only let OperationCanceledException propagate.

**5. Update XML doc comment** on the class to reflect Phase 86 adding the writer path.

**API signature reminders (from RESEARCH):**
- `CreateNamespacedLeaseAsync(body, namespaceParameter, ...)` — second param is NAMESPACE, not name. Name from body.Metadata.Name.
- `ReplaceNamespacedLeaseAsync(body, name, namespaceParameter, ...)` — second param IS the name.
- `ReadNamespacedLeaseAsync(name, namespaceParameter, ...)` — already used in reader path.
