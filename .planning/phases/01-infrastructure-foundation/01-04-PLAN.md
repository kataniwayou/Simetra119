  <action>
Create `src/SnmpCollector/Program.cs` using the Generic Host pattern (NOT WebApplication):

```csharp
using Microsoft.Extensions.Options;
using SnmpCollector.Extensions;
using SnmpCollector.Pipeline;

var builder = Host.CreateApplicationBuilder(args);

// DI registration order: Telemetry FIRST (registered first = disposed last = ForceFlush on shutdown)
builder.AddSnmpTelemetry();
builder.Services.AddSnmpConfiguration(builder.Configuration);
builder.Services.AddSnmpScheduling(builder.Configuration);

var host = builder.Build();

// Seed first correlation ID before any Quartz job fires (Pitfall 5 in research)
var correlationService = host.Services.GetRequiredService<ICorrelationService>();
correlationService.SetCorrelationId(Guid.NewGuid().ToString("N"));

try
{
    await host.RunAsync();
}
catch (OptionsValidationException ex)
{
    // Fail-fast: surface all validation failures clearly before the host accepts work
    Console.Error.WriteLine("Configuration validation failed:");
    foreach (var failure in ex.Failures)
    {
        Console.Error.WriteLine($"  - {failure}");
    }

    throw;
}
```

Key points:
- `Host.CreateApplicationBuilder(args)` -- Generic Host, NOT `WebApplication.CreateBuilder`
- Registration order: Telemetry -> Configuration -> Scheduling (matches Simetra's order)
- Correlation ID seeded BETWEEN `builder.Build()` and `host.RunAsync()` -- after DI container built, before any hosted service starts
- `OptionsValidationException` catch ensures fail-fast with human-readable error
- No health endpoints, no middleware mapping (Phase 8)
- Top-level statements (no Main method class)
