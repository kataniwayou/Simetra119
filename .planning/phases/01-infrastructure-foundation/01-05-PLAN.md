**SiteOptionsValidator.cs** -- Copy from `src/Simetra/Configuration/Validators/SiteOptionsValidator.cs` with namespace change to `SnmpCollector.Configuration.Validators`. Keep the validation logic that checks `string.IsNullOrWhiteSpace(options.Name)` and returns a failure message `"Site:Name is required"`. This is a belt-and-suspenders check alongside the `[Required]` data annotation -- the custom validator provides a clearer error message than the generic annotation message.

**OtlpOptionsValidator.cs** -- Copy from `src/Simetra/Configuration/Validators/OtlpOptionsValidator.cs` with namespace change to `SnmpCollector.Configuration.Validators`. Keep both checks:
- `string.IsNullOrWhiteSpace(options.Endpoint)` -> `"Otlp:Endpoint is required"`
- `string.IsNullOrWhiteSpace(options.ServiceName)` -> `"Otlp:ServiceName is required"`

Both validators:
- Use file-scoped namespace
- Implement `IValidateOptions<T>` with the `Validate(string? name, T options)` method
- Return `ValidateOptionsResult.Fail(failures)` when any check fails, `ValidateOptionsResult.Success` when all pass
- Are `public sealed` classes
- Use `using Microsoft.Extensions.Options;`

After creating the validator classes, add their DI registrations to `AddSnmpConfiguration` in `src/SnmpCollector/Extensions/ServiceCollectionExtensions.cs`. Add these two lines at the end of the method (after the four options registrations, before `return services`):

```csharp
// Custom IValidateOptions for cross-field validation
services.AddSingleton<IValidateOptions<SiteOptions>, SiteOptionsValidator>();
services.AddSingleton<IValidateOptions<OtlpOptions>, OtlpOptionsValidator>();
```

Also add the required `using` statement at the top of the file: `using SnmpCollector.Configuration.Validators;`
