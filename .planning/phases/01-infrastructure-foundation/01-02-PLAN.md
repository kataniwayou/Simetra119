- `global.evaluation_interval: 15s`
- NO `scrape_configs` section (pure remote_write target, no scraping)

<task type="auto">
  <name>Task 2: Create Docker Compose and Grafana provisioning</name>
  <files>
    deploy/docker-compose.yml
    deploy/grafana/provisioning/datasources/prometheus.yaml
  </files>
  <action>
Create `deploy/docker-compose.yml` with three services:

**otel-collector:**
- Image: `otel/opentelemetry-collector-contrib:0.120.0` (MUST be `-contrib`, NOT core -- `prometheusremotewrite` exporter is contrib-only)
- Volume mount: `./otel-collector-config.yaml:/etc/otelcol-contrib/config.yaml:ro`
- Ports: `4317:4317` (OTLP gRPC)
- depends_on: prometheus (collector must connect to prometheus for remote_write)

**prometheus:**
- Image: `prom/prometheus:v3.2.1`
- Volume mount: `./prometheus.yml:/etc/prometheus/prometheus.yml:ro`
- Ports: `9090:9090`
- Command (CRITICAL -- all three flags required):
  - `--config.file=/etc/prometheus/prometheus.yml`
  - `--web.enable-remote-write-receiver` (WITHOUT this flag, Prometheus rejects all remote_write pushes with HTTP 405)

**grafana:**
- Image: `grafana/grafana:latest`
- Ports: `3000:3000`
- Volume mount: `./grafana/provisioning:/etc/grafana/provisioning:ro`
- Environment:
  - `GF_AUTH_ANONYMOUS_ENABLED=true`
  - `GF_AUTH_ANONYMOUS_ORG_ROLE=Admin`
- depends_on: prometheus

Create `deploy/grafana/provisioning/datasources/prometheus.yaml`:
```yaml
apiVersion: 1
datasources:
  - name: Prometheus
    type: prometheus
    access: proxy
    url: http://prometheus:9090
    isDefault: true
    editable: false
```

This auto-provisions the Prometheus datasource on first Grafana startup.
  </action>
