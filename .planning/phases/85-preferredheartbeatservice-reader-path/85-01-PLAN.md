   - `Execute` method follows exact CorrelationJob/HeartbeatJob pattern:
     ```
     var jobKey = context.JobDetail.Key.Name;
     try { await ReadAndUpdateStampFreshnessAsync(context.CancellationToken); }
     catch (OperationCanceledException) { throw; }
     catch (Exception ex) { _logger.LogError(ex, "..."); }
     finally { _liveness.Stamp(jobKey); }
     ```
   - `ReadAndUpdateStampFreshnessAsync` private method:
     - Compute heartbeat lease name: `$"{_leaseOptions.Name}-preferred"`
     - Call `_kubeClient.CoordinationV1.ReadNamespacedLeaseAsync(heartbeatLeaseName, _leaseOptions.Namespace, cancellationToken: ct)`
     - Extract stamp time: `lease.Spec?.RenewTime ?? lease.Spec?.AcquireTime`
     - If stampTime is null: `_preferredLeaderService.UpdateStampFreshness(false); return;`
     - Normalize UTC: `DateTime.SpecifyKind(stampTime.Value, DateTimeKind.Utc)` before comparison
     - Compute threshold: `TimeSpan.FromSeconds(_leaseOptions.DurationSeconds + 5)`
     - Compute age: `DateTime.UtcNow - normalizedStampTime`
     - Call `_preferredLeaderService.UpdateStampFreshness(age <= threshold)`
     - Catch `k8s.Autorest.HttpOperationException` with `when (ex.Response.StatusCode == System.Net.HttpStatusCode.NotFound)`: call `UpdateStampFreshness(false)` — 404 = stale
     - Catch `OperationCanceledException`: rethrow
     - Catch generic `Exception`: log Warning "transient K8s API error reading lease {LeaseName} — keeping last value", do NOT update freshness (keep-last-value semantics)
