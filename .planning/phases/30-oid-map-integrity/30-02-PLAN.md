<objective>
Add duplicate OID key and duplicate metric name validation to OidMapWatcherService, running before UpdateMap is called.

Purpose: Operators need clear signals when their oidmaps.json contains duplicate OID keys or duplicate metric name values. Without validation, Dictionary deserialization silently drops duplicate keys (last-write-wins) and duplicate names silently coexist. The "skip both" policy ensures ambiguous entries never enter the map.
Output: OidMapWatcherService with JsonDocument-based validation, structured warning logs, and unit tests for the validation logic.
</objective>

  <action>
Replace the `HandleConfigMapChangedAsync` JSON parsing section. The current code uses `JsonSerializer.Deserialize<Dictionary<string, string>>` which silently drops duplicate keys. Replace with a `ValidateAndParseOidMap` method that uses `JsonDocument` for explicit property enumeration.

1. Add a new internal static method `ValidateAndParseOidMap` (internal for testability) that takes `string jsonContent` and `ILogger logger` and returns `Dictionary<string, string>?`. The method:

   a. Parse JSON with `JsonDocument.Parse(jsonContent, new JsonDocumentOptions { CommentHandling = JsonCommentHandling.Skip, AllowTrailingCommas = true })`. These options match the existing `JsonOptions` and are required because `oidmaps.json` uses `//` comments and trailing commas.

   b. **Pass 1 — Enumerate all properties, detect duplicate OID keys:**
      - Create `HashSet<string>(StringComparer.OrdinalIgnoreCase) seen` and `HashSet<string>(StringComparer.OrdinalIgnoreCase) duplicateOids` (OrdinalIgnoreCase matches `MergeWithHeartbeatSeed` behavior).
      - Create `List<(string oid, string name)> rawEntries`.
      - Iterate `doc.RootElement.EnumerateObject()`. For each property:
        - Get `oid = property.Name`, `name = property.Value.GetString()`.
        - If `name` is null or empty, log warning `"OidMap entry skipped: {Oid} has null or empty metric name"` and skip.
        - If `!seen.Add(oid)`, add to `duplicateOids` (do NOT add to rawEntries — but the first occurrence IS in rawEntries already, and will be filtered out in step d).
        - Else add `(oid, name)` to `rawEntries`.
      - For each oid in `duplicateOids`, log: `"OidMap duplicate OID key detected: {Oid} appears multiple times -- all entries for this OID will be skipped"`.

   c. **Pass 2 — Detect duplicate metric name values:**
      - Create `Dictionary<string, int>(StringComparer.Ordinal) nameCounts` (Ordinal — metric names are case-sensitive).
      - Iterate `rawEntries` and count each name.
      - Build `HashSet<string>(StringComparer.Ordinal) duplicateNames` from names with count > 1.
      - For each duplicate name, log: `"OidMap duplicate metric name detected: {MetricName} maps to multiple OIDs -- all entries for this name will be skipped"`.

   d. **Pass 3 — Build clean dictionary, filtering out both categories:**
      - Create `Dictionary<string, string>(StringComparer.OrdinalIgnoreCase) cleanEntries`.
      - Iterate `rawEntries`. Skip if `duplicateOids.Contains(oid)` — log per entry: `"OidMap duplicate OID key skipped: {Oid} -> {MetricName}"`.
      - Skip if `duplicateNames.Contains(name)` — log per entry: `"OidMap duplicate metric name skipped: {Oid} -> {MetricName}"`.
      - Otherwise add to `cleanEntries`.

   e. **Empty check:** If `cleanEntries.Count == 0`, log at ERROR: `"OidMap validation produced empty map -- all entries were duplicates or invalid"`.

   f. Return `cleanEntries`.

2. In `HandleConfigMapChangedAsync`, replace the `JsonSerializer.Deserialize` block (lines 169-188 approximately) with a call to `ValidateAndParseOidMap(jsonContent, _logger)`. If it returns null, return early (the method should return null only on parse failure, wrapping the JsonException in a try/catch inside the method). The rest of HandleConfigMapChangedAsync (semaphore, UpdateMap call, logging) stays the same.

3. The `JsonOptions` static field can be removed since it was only used by `JsonSerializer.Deserialize`. If it's referenced elsewhere, keep it. Check before removing.

IMPORTANT: Do NOT validate inside OidMapService. Validation belongs in OidMapWatcherService per locked decisions. OidMapService receives only clean, pre-validated entries.

IMPORTANT: The `rawEntries` list from Pass 1 contains the FIRST occurrence of every OID. Entries whose OID is in `duplicateOids` will be removed in Pass 3. This correctly handles Pitfall 1 (both occurrences skipped).
