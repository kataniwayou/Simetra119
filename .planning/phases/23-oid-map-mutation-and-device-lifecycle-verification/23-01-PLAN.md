**1. `tests/e2e/scenarios/18-oid-rename.sh` (MUT-01: OID Rename)**

```
# Scenario 18: OID map rename propagates to Prometheus
# Renames e2e_gauge_test -> e2e_renamed_gauge in oidmaps ConfigMap,
# verifies new metric_name appears in snmp_gauge, then restores.
```

Flow:
1. `snapshot_configmaps`
2. `kubectl apply -f "$SCRIPT_DIR/fixtures/oid-renamed-configmap.yaml" -n simetra`
3. Poll with 60s deadline, 3s interval for `snmp_gauge{device_name="E2E-SIM",metric_name="e2e_renamed_gauge"}` to have result count > 0
4. If found: extract `metric_name`, `oid`, `device_name` labels. Verify:
   - `metric_name` = `e2e_renamed_gauge`
   - `oid` = `1.3.6.1.4.1.47477.999.1.1.0`
   - `device_name` = `E2E-SIM`
   - Record pass with evidence string
5. If not found within timeout: record fail
6. `restore_configmaps`
7. Poll with 60s deadline for `snmp_gauge{device_name="E2E-SIM",metric_name="e2e_gauge_test"}` to reappear (confirms restoration)
8. Log restoration confirmation

**2. `tests/e2e/scenarios/19-oid-remove.sh` (MUT-02: OID Removal -> Unknown)**

# Scenario 19: OID removal causes metric to become Unknown

**3. `tests/e2e/scenarios/20-oid-add.sh` (MUT-03: OID Addition)**

```
# Scenario 20: OID addition causes Unknown OID to get correct metric_name
# Two-step mutation: (1) add unmapped OIDs to E2E-SIM device config so they
# are polled as Unknown, (2) add oidmap entry so .999.2.1.0 gets named.
```

Flow:
1. `snapshot_configmaps`
2. **Step 1 -- Start polling unmapped OIDs:**
   `kubectl apply -f "$SCRIPT_DIR/fixtures/e2e-sim-unmapped-configmap.yaml" -n simetra`
3. Poll with 60s deadline for `snmp_gauge{device_name="E2E-SIM",metric_name="Unknown",oid="1.3.6.1.4.1.47477.999.2.1.0"}` to appear (proves it's being polled but unmapped)
4. If Unknown not found within timeout: record fail and skip to restore
5. **Step 2 -- Add OID mapping:**
   `kubectl apply -f "$SCRIPT_DIR/fixtures/oid-added-configmap.yaml" -n simetra`
6. Poll with 60s deadline for `snmp_gauge{device_name="E2E-SIM",metric_name="e2e_unmapped_gauge"}` to appear
7. If found: extract labels, verify `metric_name` = `e2e_unmapped_gauge`, `oid` = `1.3.6.1.4.1.47477.999.2.1.0`, `device_name` = `E2E-SIM`
8. Record pass/fail with evidence
9. `restore_configmaps`
10. Log restoration confirmation
