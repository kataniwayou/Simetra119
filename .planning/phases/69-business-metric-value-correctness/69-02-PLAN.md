    1. Sets SCENARIO_NAME: `"MVC-08: snmp_gauge value updates after simulator change (42->99)"`
    2. Overrides e2e_gauge_test (OID suffix "1.1") to value 99: `sim_set_oid "1.1" "99"`
    3. Polls Prometheus in a loop with 40s deadline and 3s interval:
       - Query: `snmp_gauge{device_name="E2E-SIM",resolved_name="e2e_gauge_test"}`
       - Extract value with `jq -r '.data.result[0].value[1]'`, then `cut -d. -f1`
       - If value equals "99", break with FOUND=1
    4. ALWAYS calls `reset_oid_overrides` after assertions (cleanup). This is critical -- if not cleaned up, subsequent scenarios (including scenario 11 and scenario 86) expecting value=42 will fail.
    5. If FOUND=1: `record_pass` with evidence showing the final value.
    6. If FOUND=0: `record_fail` with evidence showing the last observed value.

    Do NOT add shebang or `set -euo pipefail` -- scenario file is sourced by run-all.sh.

    IMPORTANT timing: The poll interval for .999.1.x is 10s. With OTel export lag up to 15s and Prometheus scrape up to 5s, worst case is ~30s. The 40s deadline with 3s poll interval provides comfortable margin.

    IMPORTANT cleanup order: Call `reset_oid_overrides` AFTER the poll loop ends and BEFORE record_pass/record_fail. This ensures cleanup happens regardless of pass/fail. Do NOT use a trap handler -- just place the cleanup call between the loop and the assertion.

    Follow this exact structure:
    ```bash
    # Scenario 93: MVC-08 -- snmp_gauge value updates after simulator change
    SCENARIO_NAME="MVC-08: snmp_gauge value updates after simulator change (42->99)"

    # Override Gauge32 OID (.999.1.1) to 99
    sim_set_oid "1.1" "99"

    # Poll Prometheus until value reflects 99 (40s deadline, 3s interval)
    DEADLINE=$(( $(date +%s) + 40 ))
    FOUND=0
    FINAL_VALUE=""
    while [ "$(date +%s)" -lt "$DEADLINE" ]; do
        RESULT=$(query_prometheus 'snmp_gauge{device_name="E2E-SIM",resolved_name="e2e_gauge_test"}') || true
        VAL=$(echo "$RESULT" | jq -r '.data.result[0].value[1]' 2>/dev/null) || VAL=""
        VAL_INT=$(echo "$VAL" | cut -d. -f1)
        if [ "$VAL_INT" = "99" ]; then
            FOUND=1
            FINAL_VALUE="$VAL"
            break
        fi
        FINAL_VALUE="$VAL"
        sleep 3
    done

    # Cleanup: clear override so subsequent scenarios see value=42
    reset_oid_overrides

    # Assert
    if [ "$FOUND" -eq 1 ]; then
        record_pass "$SCENARIO_NAME" "value updated to 99 within poll cycle. final_value=${FINAL_VALUE}"
    else
        record_fail "$SCENARIO_NAME" "value did not update to 99 within 40s. last_value=${FINAL_VALUE}"
    fi
    ```

    This produces exactly 1 SCENARIO_RESULTS entry at runtime (index 95 in the 0-based array), which falls within the "Business Metric Value Correctness|88|95" category range set by Plan 01.
