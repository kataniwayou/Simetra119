  <action>
Create `simulators/e2e-sim/e2e_simulator.py` by copy-adapting from `simulators/obp/obp_simulator.py`. This is a much simpler simulator -- all values are static (no random walk, no state toggling).

**Structure (follow OBP simulator structure exactly):**

1. **Imports** -- identical to OBP: `pysnmp.entity.engine`, `config`, `cmdrsp`, `context`, `carrier.asyncio.dgram.udp`, `proto.api.v2c`, `hlapi.v3arch.asyncio` (SnmpEngine as HlapiEngine, CommunityData, UdpTransportTarget, ContextData, send_notification, NotificationType, ObjectIdentity). Plus `asyncio`, `logging`, `os`, `signal`, `socket`.

2. **Configuration env vars:**
   - `DEVICE_NAME` (default: `E2E-SIM`)
   - `COMMUNITY` (default: `Simetra.{DEVICE_NAME}`)
   - `TRAP_TARGET` (default: `simetra-pods.simetra.svc.cluster.local`) -- IMPORTANT: use simetra-pods headless service, NOT snmp-collector ClusterIP
   - `TRAP_PORT` (default: `10162`) -- IMPORTANT: collector listens on 10162, NOT 162
   - `TRAP_INTERVAL` (default: `30`) -- seconds between valid traps
   - `BAD_TRAP_INTERVAL` (default: `45`) -- seconds between bad-community traps (offset from valid to avoid simultaneous sends)
   - `STARTUP_DELAY = 12` -- constant, seconds before first trap

3. **OID definitions** -- use `E2E_PREFIX = "1.3.6.1.4.1.47477.999"`:

   Mapped OIDs (7 total, subtree .999.1.x):
   ```
   (.999.1.1, "gauge_test",     v2c.Gauge32,      42)
   (.999.1.2, "integer_test",   v2c.Integer32,    100)
   (.999.1.3, "counter32_test", v2c.Counter32,   5000)
   (.999.1.4, "counter64_test", v2c.Counter64,   1000000)
   (.999.1.5, "timeticks_test", v2c.TimeTicks,   360000)
   (.999.1.6, "info_test",      v2c.OctetString, "E2E-TEST-VALUE")
   (.999.1.7, "ip_test",        v2c.IpAddress,   "10.0.0.1")
   ```

   Unmapped OIDs (2 total, subtree .999.2.x):
   ```
   (.999.2.1, "unmapped_gauge", v2c.Gauge32,     99)
   (.999.2.2, "unmapped_info",  v2c.OctetString, "UNMAPPED")
   ```

4. **SNMP engine setup** -- identical pattern to OBP:
   - `engine.SnmpEngine()`, `config.add_transport` on `0.0.0.0:161`
   - `config.add_v1_system(snmpEngine, "my-area", COMMUNITY)`
   - `config.add_vacm_user(snmpEngine, 2, "my-area", "noAuthNoPriv", (1, 3, 6, 1, 4, 1))`
   - All 4 command responders (Get, Next, Bulk, Set)
   - `mibBuilder` from `snmpContext.get_mib_instrum().get_mib_builder()`
   - Import `MibScalar, MibScalarInstance`

5. **DynamicInstance class** -- identical to OBP. Takes `oid_tuple, index_tuple, syntax, get_value_fn`. `getValue` returns `self.getSyntax().clone(self._get_value_fn())`.

6. **OID registration loop** -- iterate ALL_OIDS (mapped + unmapped), register MibScalar + DynamicInstance pairs with `lambda v=static_value: v` callbacks. Export with `mibBuilder.export_symbols("__E2E-SIM-MIB", **symbols)`.

7. **Helper functions:**
   - `oid_str_to_tuple(oid_str)` -- same as OBP
   - `supervised_task(name, coro_fn)` -- same as OBP (exponential backoff restart)
   - `resolve_trap_targets(hostname)` -- same as OBP (socket.getaddrinfo)

8. **Trap sending:**
   - `TRAP_OID = f"{E2E_PREFIX}.3.1"` -- notification OID
   - `GAUGE_OID = f"{E2E_PREFIX}.1.1.0"` -- varbind using gauge value
   - `send_trap_to_targets(community_string)` -- resolve targets, send via hlapi send_notification with NotificationType(ObjectIdentity(TRAP_OID)).add_varbinds((GAUGE_OID, v2c.Gauge32(42)))
   - `valid_trap_loop()` -- sleep STARTUP_DELAY, then loop: sleep TRAP_INTERVAL, send_trap_to_targets(COMMUNITY)
   - `bad_community_trap_loop()` -- sleep STARTUP_DELAY, then loop: sleep BAD_TRAP_INTERVAL, send_trap_to_targets("BadCommunity")

9. **main() function:**
   - Log startup info (OID count, community, config, registered OIDs, trap OID)
   - Create asyncio event loop
   - Create 2 supervised tasks: valid_trap_loop, bad_community_trap_loop
   - Signal handlers (SIGTERM, SIGINT) cancel tasks + close dispatcher
   - `snmpEngine.open_dispatcher()` to start

10. **Logging format:** `%(asctime)s [E2E-SIM] %(message)s`

**What NOT to do:**
- No random walk, no state mutation, no power updates -- all values are static constants
- No per-link loops -- there are no links, just flat OIDs
- No HTTP API -- interval-only traps per CONTEXT decision
- No IpAddress import needed separately -- `v2c.IpAddress` is available from `pysnmp.proto.api.v2c`
