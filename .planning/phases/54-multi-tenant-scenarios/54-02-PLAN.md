Create `tests/e2e/scenarios/35-mts-02-advance-gate.sh` following Phase 53 script structure. This script has TWO assertion windows in sequence within one script (same pattern as STS-04 suppression window which had multiple assertion phases).

**Header comment:** "Scenario 35: MTS-02 Advance gate — P1 Commanded blocks P2 (02A), then P1 suppressed (ConfirmedBad) passes gate for P2 (02B)"

Explain the timing model in the header comment:
- Uses tenant-cfg03-two-diff-prio-mts.yaml (P1 SuppressionWindowSeconds=30, P2 SuppressionWindowSeconds=10)
- T=0: command_trigger set, P1 first cycle → Commanded (sent=1), gate blocks P2
- T=15s: next SnapshotJob cycle, P1's 30s window still active → suppressed → ConfirmedBad (enqueueCount==0) → gate passes → P2 evaluated → P2 Commanded (sent)

**Structure:**

1. **FIXTURES_DIR** (standard)

2. **Setup:**
   - `save_configmap "simetra-tenants" "simetra" "$FIXTURES_DIR/.original-tenants-configmap.yaml" || true`
   - Apply the MTS fixture: `kubectl apply -f "$FIXTURES_DIR/tenant-cfg03-two-diff-prio-mts.yaml" > /dev/null 2>&1 || true`
   - `poll_until_log 60 5 "Tenant vector reload complete\|TenantVectorWatcher initial load complete" 30` (with warn on timeout)

3. **Scenario switch:**
   - `sim_set_scenario command_trigger`

4. **Baseline counters (before MTS-02A window):**
   - `BEFORE_SENT=$(snapshot_counter "snmp_command_sent_total" 'device_name="E2E-SIM"')`

5. **=== MTS-02A: Gate Blocked ===**

   5a. **Positive log assertion — P1 tier=4:**
   - `poll_until_log 90 5 "e2e-tenant-P1.*tier=4 — commands enqueued" 60`
   - On success: `record_pass "MTS-02A: P1 tier=4 detected (gate blocker)" "log=tier4_P1_found"`
   - On failure: `record_fail "MTS-02A: P1 tier=4 detected (gate blocker)" "tier4 log for P1 not found within 90s"`

   5b. **Negative log assertion — P2 must NOT appear in tier logs:**
   After confirming P1's tier=4 log, check that P2 has no tier logs in the recent window.
   ```bash
   PODS=$(kubectl get pods -n simetra -l app=snmp-collector \
       -o jsonpath='{.items[*].metadata.name}' 2>/dev/null) || true
   P2_FOUND=0
   for POD in $PODS; do
       P2_LOGS=$(kubectl logs "$POD" -n simetra --since=30s 2>/dev/null \
           | grep "e2e-tenant-P2.*tier=" || echo "") || true
       if [ -n "$P2_LOGS" ]; then
           P2_FOUND=1
           break
       fi
   done
   ```
   - If `P2_FOUND == 0`: `record_pass "MTS-02A: P2 not evaluated when gate blocked" "e2e-tenant-P2 tier log absent in 30s window"`
   - If `P2_FOUND == 1`: `record_fail "MTS-02A: P2 not evaluated when gate blocked" "e2e-tenant-P2 tier log found unexpectedly: $P2_LOGS"`

   5c. **Counter assertion — sent counter delta >= 1 (P1 sent, proving P1 was evaluated):**
   - `AFTER_SENT_A=$(snapshot_counter "snmp_command_sent_total" 'device_name="E2E-SIM"')`
   - `DELTA_A=$((AFTER_SENT_A - BEFORE_SENT))`
   - `assert_delta_gt "$DELTA_A" 0 "MTS-02A: P1 sent counter incremented" "sent_delta=${DELTA_A} $(get_evidence "snmp_command_sent_total" 'device_name="E2E-SIM"')"`

6. **=== MTS-02B: Gate Passed ===**

   The script does NOT reset_scenario between 02A and 02B. The command_trigger scenario stays active. The key is timing: at the next SnapshotJob cycle (T=15s from first command), P1's 30s suppression window is still active. P1 reaches Tier 4 but all commands are suppressed (enqueueCount==0), so P1 returns ConfirmedBad. The gate passes. P2 is evaluated, reaches Tier 4 (first time for P2, no suppression), and sends commands.

   6a. **Baseline for MTS-02B window:**
   - `BEFORE_SENT_B=$(snapshot_counter "snmp_command_sent_total" 'device_name="E2E-SIM"')`

   6b. **Positive log assertion — P2 tier=4:**
   - `poll_until_log 60 5 "e2e-tenant-P2.*tier=4 — commands enqueued" 30`
   - On success: `record_pass "MTS-02B: P2 tier=4 detected (gate passed)" "log=tier4_P2_found"`
   - On failure: `record_fail "MTS-02B: P2 tier=4 detected (gate passed)" "tier4 log for P2 not found within 60s"`

   6c. **Counter assertion — sent counter incremented for P2:**
   - `AFTER_SENT_B=$(snapshot_counter "snmp_command_sent_total" 'device_name="E2E-SIM"')`
   - `DELTA_B=$((AFTER_SENT_B - BEFORE_SENT_B))`
   - `assert_delta_gt "$DELTA_B" 0 "MTS-02B: Sent counter incremented after gate passed" "sent_delta=${DELTA_B} $(get_evidence "snmp_command_sent_total" 'device_name="E2E-SIM"')"`

7. **Cleanup:**
   - `reset_scenario`
   - Restore original ConfigMap (standard pattern)

**Key details:**
- The em-dash in log patterns is U+2014 (must match SnapshotJob.cs exactly)
- The negative log assertion uses `--since=30s` to scope to the recent window only (avoids false positives from prior scenarios)
- MTS-02A and MTS-02B flow naturally in one script because 02B happens on the NEXT SnapshotJob cycle after 02A — no scenario reset needed between them
- All counter assertions use `device_name="E2E-SIM"` (no pod label filter) — this satisfies the "sum() without pod label filter" requirement
- Use `log_info "MTS-02: ..."` or `log_info "MTS-02A: ..."` / `log_info "MTS-02B: ..."` for all log lines
- The `|| true` and `|| echo ""` guards must be present on kubectl commands to prevent set -e failures
