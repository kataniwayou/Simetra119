**Structure:**

1. **Header comment:** "Scenario 34: MTS-01 Same-priority independence — both P1 tenants produce independent tier=4 log lines and command counters"

2. **FIXTURES_DIR:** `FIXTURES_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)/fixtures"`

3. **Setup:**
   - `save_configmap "simetra-tenants" "simetra" "$FIXTURES_DIR/.original-tenants-configmap.yaml" || true`
   - `kubectl apply -f "$FIXTURES_DIR/tenant-cfg02-two-same-prio.yaml" > /dev/null 2>&1 || true`
   - `poll_until_log 60 5 "Tenant vector reload complete\|TenantVectorWatcher initial load complete" 30` (with warn on timeout)

4. **Scenario switch:**
   - `sim_set_scenario command_trigger`

5. **Baseline counter:**
   - `BEFORE_SENT=$(snapshot_counter "snmp_command_sent_total" 'device_name="E2E-SIM"')`

6. **Sub-scenario 34a: e2e-tenant-A tier=4 log:**
   - `poll_until_log 90 5 "e2e-tenant-A.*tier=4 — commands enqueued" 60`
   - On success: `record_pass "MTS-01: e2e-tenant-A tier=4 log detected" "log=tier4_tenantA_found"`
   - On failure: `record_fail "MTS-01: e2e-tenant-A tier=4 log detected" "tier4 log for e2e-tenant-A not found within 90s"`

7. **Sub-scenario 34b: e2e-tenant-B tier=4 log:**
   - `poll_until_log 60 5 "e2e-tenant-B.*tier=4 — commands enqueued" 30` (shorter timeout — scenario already active)
   - On success: `record_pass "MTS-01: e2e-tenant-B tier=4 log detected" "log=tier4_tenantB_found"`
   - On failure: `record_fail "MTS-01: e2e-tenant-B tier=4 log detected" "tier4 log for e2e-tenant-B not found within 60s"`

8. **Sub-scenario 34c: counter delta >= 2:**
   - `AFTER_SENT=$(snapshot_counter "snmp_command_sent_total" 'device_name="E2E-SIM"')`
   - `DELTA=$((AFTER_SENT - BEFORE_SENT))`
   - `assert_delta_gt "$DELTA" 1 "MTS-01: Both tenants commanded — sent delta >= 2" "sent_delta=${DELTA} $(get_evidence "snmp_command_sent_total" 'device_name="E2E-SIM"')"`
   - Note: `assert_delta_gt "$DELTA" 1` means delta > 1, i.e., delta >= 2. This requires both tenants to have sent commands.

9. **Cleanup:**
   - `reset_scenario`
   - Restore original ConfigMap (same pattern as Phase 53 scripts)

**Key details:**
- Use `log_info "MTS-01: ..."` for all log lines (match Phase 53 prefix style)
- Use `> /dev/null 2>&1` on kubectl apply
- The em-dash in "tier=4 — commands enqueued" is U+2014 (match exact SnapshotJob log string)
- Do NOT use `sum()` in snapshot_counter — the function already queries the raw metric; `sum()` is implicit in Prometheus instant queries for a single metric name
- All counter assertions use `device_name="E2E-SIM"` (not per-tenant) since `snmp_command_sent_total` aggregates all tenants under this device label
