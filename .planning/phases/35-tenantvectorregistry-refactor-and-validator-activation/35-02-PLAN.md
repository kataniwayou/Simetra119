## Objective

Move all per-entry validation, IP resolution, OidMap checks, DeviceRegistry checks, and TEN-13 completeness gate from `TenantVectorRegistry.Reload()` into `TenantVectorWatcherService`. After this plan, TenantVectorRegistry is a pure data store: it receives pre-validated `TenantVectorOptions` (all invalid entries already filtered out, IPs already resolved) and builds routing index + priority groups. TenantVectorWatcherService owns all cross-service dependencies (IOidMapService, IDeviceRegistry).

This completes requirements TEN-04 (remove IOidMapService from registry), TEN-06 (CommandName handling), TEN-08 (validator activation), CLN-01 (delete ResolveIp/DeriveIntervalSeconds), CLN-02 (remove IOidMapService + IDeviceRegistry from constructor).

### Task 1: Create `ValidateAndBuildTenants` in TenantVectorWatcherService + refactor TenantVectorRegistry to pure store

**Add `ValidateAndBuildTenants` to TenantVectorWatcherService:**

Add an `internal static TenantVectorOptions ValidateAndBuildTenants(TenantVectorOptions options, IOidMapService oidMapService, IDeviceRegistry deviceRegistry, ILogger logger)` method. Synchronous (all lookups are in-memory).

This method returns a new `TenantVectorOptions` with filtered/resolved data — invalid entries removed, IPs resolved. The registry's `Reload(TenantVectorOptions)` signature stays the same but now trusts the input is clean.

Move ALL validation logic from `TenantVectorRegistry.Reload()` into this method:

**Per-metric entry validation (move from TenantVectorRegistry lines 113-193):**
1. Structural: empty Ip → Error log + skip
2. Structural: port out of range [1, 65535] → Error log + skip
3. Structural: empty MetricName → Error log + skip
4. Role validation: must be "Evaluate" or "Resolved" → Error log + skip
5. MetricName resolution (TEN-05): `oidMapService.ContainsMetricName(metric.MetricName)` → Error log + skip
6. IP+Port existence (TEN-07): `deviceRegistry.TryGetByIpPort(metric.Ip, metric.Port, out _)` → Error log + skip

After passing all checks:
- **Resolve IP** (replaces `TenantVectorRegistry.ResolveIp`): iterate `deviceRegistry.AllDevices` to find device where `device.ConfigAddress` equals `metric.Ip` (case-insensitive), use `device.ResolvedIp`. If not found, keep original IP. Write the resolved IP back into a new `MetricSlotOptions` copy with the resolved IP (or mutate if the class allows it — check if `MetricSlotOptions.Ip` has a setter).

  Since `MetricSlotOptions` has `{ get; set; }` properties (it's a config POCO), the watcher can directly set `metric.Ip = resolvedIp` on the options object before passing to the registry. This avoids creating new objects.

**Per-command entry validation (move from TenantVectorRegistry lines 196-256):**
1. Structural: empty Ip → Error log + skip
2. Structural: port out of range → Error log + skip
3. Structural: empty CommandName → Error log + skip
4. ValueType validation (TEN-03): must be "Integer32", "IpAddress", or "OctetString" → Error log + skip
5. Empty Value → Error log + skip
6. IP+Port existence (TEN-07): `deviceRegistry.TryGetByIpPort` → Error log + skip

**TEN-06 CommandName handling:** After the structural checks pass (CommandName is non-empty), the watcher does NOT check if the CommandName exists in any command map. Unresolvable CommandNames are stored as-is — resolution is deferred to execution time. If you want a Debug log for observability, add: `logger.LogDebug("Tenant '{TenantId}' Commands[{Index}]: CommandName '{CommandName}' stored as-is (resolution deferred to execution)", tenantId, k, cmd.CommandName);`

**TEN-13 completeness gate (move from TenantVectorRegistry lines 258-270):**
- Count surviving Resolved metrics, Evaluate metrics, and commands
- If any count is 0 → Error log + skip entire tenant
- Same log message template as current

**Building the cleaned TenantVectorOptions:**
```csharp
var cleanTenants = new List<TenantOptions>();
for (var i = 0; i < options.Tenants.Count; i++)
{
    var tenantOpts = options.Tenants[i];
    var tenantId = !string.IsNullOrWhiteSpace(tenantOpts.Name) ? tenantOpts.Name : $"tenant-{i}";

    // Filter metrics (validation + IP resolution)
    var cleanMetrics = new List<MetricSlotOptions>();
    int evaluateCount = 0, resolvedCount = 0;
    for (var j = 0; j < tenantOpts.Metrics.Count; j++) { /* validation loop */ }

    // Filter commands
    var cleanCommands = new List<CommandSlotOptions>();
    for (var k = 0; k < tenantOpts.Commands.Count; k++) { /* validation loop */ }

    // TEN-13 gate
    if (resolvedCount == 0 || evaluateCount == 0 || cleanCommands.Count == 0) { /* Error + skip */ continue; }

    // Build clean tenant with filtered lists
    cleanTenants.Add(new TenantOptions
    {
        Name = tenantOpts.Name,
        Priority = tenantOpts.Priority,
        Metrics = cleanMetrics,
        Commands = cleanCommands
    });
}
return new TenantVectorOptions { Tenants = cleanTenants };
```
