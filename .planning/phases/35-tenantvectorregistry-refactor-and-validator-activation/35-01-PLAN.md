## Objective

Move all validation, DNS resolution, CommunityString extraction, duplicate detection, and OID resolution logic from `DeviceRegistry` into `DeviceWatcherService`. After this plan, DeviceRegistry is a pure data store: it accepts `List<DeviceInfo>` and builds FrozenDictionary indexes. DeviceWatcherService owns all validation and produces clean `List<DeviceInfo>` before calling the registry.

This follows the "watcher validates, registry stores" pattern already established by OidMapWatcherService and CommandMapWatcherService.

**Add `ValidateAndBuildDevicesAsync` to DeviceWatcherService:**

Add an `internal static async Task<List<DeviceInfo>> ValidateAndBuildDevicesAsync(List<DeviceOptions> devices, IOidMapService oidMapService, ILogger logger, CancellationToken ct)` method. This mirrors the `CommandMapWatcherService.ValidateAndParseCommandMap` pattern (internal static, explicit parameters, callable from Program.cs).

Move ALL validation logic from `DeviceRegistry` constructor and `ReloadAsync` into this method:

1. **CommunityString extraction** — `CommunityStringHelper.TryExtractDeviceName(d.CommunityString, out var deviceName)`. Invalid → Error log + skip (continue). Use exact same log message templates from DeviceRegistry.
2. **DNS resolution (async)** — `IPAddress.TryParse` for IPs, `Dns.GetHostAddressesAsync` for hostnames. Use `addresses.First(a => a.AddressFamily == AddressFamily.InterNetwork)` and `ip.MapToIPv4()`. This is the async part.
3. **BuildPollGroups** — move the `BuildPollGroups` private method from DeviceRegistry into this static method (or as a nested static helper). It calls `oidMapService.ResolveToOid(name)` for each MetricName. Preserve exact same log messages (Warning for unresolvable names, Info for resolution summary, Warning for zero-OID groups).
4. **Build DeviceInfo** — `new DeviceInfo(deviceName, d.IpAddress, ip.ToString(), d.Port, pollGroups, d.CommunityString)`.
5. **Duplicate IP+Port** — check `byIpPortSeen` dictionary. Duplicate → Error log + skip (continue). Same log templates.
6. **Duplicate CommunityString** — check `seenCommunityStrings` dictionary. Same IP+Port would already be caught by step 5, so this only fires for different IP+Port. Warning log, both load. Same log templates.

The method returns `List<DeviceInfo>` — clean, validated, DNS-resolved, poll-groups-built.
