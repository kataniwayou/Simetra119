Three sub-fixes:

**A. Fix PSS-18c flaky assertion** (66-pss-18-g1-resolved-unresolved.sh, line 105):
The bug: `sleep 10` observation window but `--since=12s` log query. The 2s extra (12-10) creates an overlap where prior-cycle G2 tier logs from before the gate-block can bleed in, causing false positive "G2 tier log found".

Fix: Change `--since=12s` to `--since=10s` on line 105. The stabilization poll (`poll_until_log ... "2 evaluated"`) already guarantees the gate is blocking before the observation window starts, so there is no need for the --since to exceed the sleep. The --since should exactly match the sleep to avoid capturing pre-observation-window logs.

Also update the record_pass message on line 115 from `"G2 tier logs absent in 15s window"` to `"G2 tier logs absent in 10s window"` (cosmetic accuracy).

**B. Fix PSS-19c flaky assertion** (67-pss-19-g1-healthy-unresolved.sh, line 103):
Identical fix: change `--since=12s` to `--since=10s` on line 103. Also update record_pass message on line 113 from `"G2 tier logs absent in 15s window"` to `"G2 tier logs absent in 10s window"`.

**C. Fix standalone runner report category**: When run-stage2.sh or run-stage3.sh runs standalone, SCENARIO_RESULTS indices start at 0 (not 52). The PSS category in _REPORT_CATEGORIES maps to indices 52-67, so it gets skipped (`start_idx >= total` -> `continue`).

Fix approach: Have each standalone runner override `_REPORT_CATEGORIES` before calling `generate_report`. This is clean because `_REPORT_CATEGORIES` is a global variable and `generate_report` reads it at call time.

In **run-stage2.sh**, add these lines immediately before the `generate_report` call at line 146 (and also before the early-exit generate_report at line 111):
```bash
# Override report categories for standalone PSS Stage 2 runner
_REPORT_CATEGORIES=(
    "PSS Stage 1: Single Tenant Evaluation|0|5"
    "PSS Stage 2: Two Tenant Independence|6|8"
)
```

In **run-stage3.sh**, add these lines immediately before the `generate_report` call at line 246 (and also before the two early-exit generate_report calls at lines 124 and 161):
```bash
# Override report categories for standalone PSS Stage 3 runner
_REPORT_CATEGORIES=(
    "PSS Stage 1: Single Tenant Evaluation|0|5"
    "PSS Stage 2: Two Tenant Independence|6|8"
    "PSS Stage 3: Advance Gate Logic|9|15"
)
```

Do NOT modify _REPORT_CATEGORIES in report.sh itself -- the default definition there (indices 0-67 across all categories) is correct for run-all.sh which runs the full suite.

Do NOT change any assertion logic, scenario structure, sleep durations, stabilization polls, or metric snapshot patterns.
