    Create `tests/e2e/lib/sim.sh` following the exact patterns established in kubectl.sh and prometheus.sh. The file must use `#!/usr/bin/env bash` and `set -euo pipefail` (matching existing lib files).

    **Functions to implement:**

    1. `sim_set_scenario <name>` -- POST to `http://localhost:8080/scenario/{name}`. Use `curl -sf -o /dev/null -w '%{http_code}'` pattern. Log via `log_info` (from common.sh). Return 1 on curl failure or non-200 HTTP code. Log the scenario name on success.

    2. `reset_scenario` -- Convenience wrapper: calls `sim_set_scenario default`. This is the belt-and-suspenders reset called at the start of every snapshot scenario.

    3. `get_active_scenario` -- GET `http://localhost:8080/scenario`. Use `curl -sf` to fetch JSON, extract the scenario name with `jq -r '.scenario'` (the HTTP endpoint returns `{"scenario": "name"}`). Print the name to stdout. Return 1 on failure.

    4. `poll_until_log <timeout_s> <interval_s> <grep_pattern> [since_seconds]` -- Search all snmp-collector pod logs for a grep pattern. Implementation:
       - Get pod list via `kubectl get pods -n simetra -l app=snmp-collector -o jsonpath='{.items[*].metadata.name}'`
       - Loop until deadline: for each pod, `kubectl logs $POD -n simetra --since=${since}s`, pipe through `grep "$pattern" > /dev/null 2>&1` (NOT `grep -q` -- avoid SIGPIPE under pipefail, matching scenario 28 pattern)
       - Return 0 on first match (any pod), return 1 on timeout
       - Default `since` parameter: 60 seconds

    **Global variable:** `SIM_URL="http://localhost:8080"` at top of file.

    **Anti-patterns to avoid:**
    - Do NOT use `grep -q` (SIGPIPE issue with pipefail, per scenario 28 pattern)
    - Do NOT add a shebang that differs from other lib files
    - Do NOT add functions beyond the 4 listed (keep library minimal)
