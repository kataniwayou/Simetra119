  <action>
Create `tests/e2e/lib/sim_command.sh` as a standalone executable Bash script. The research at `83-RESEARCH.md` contains a complete verified skeleton — use it as the foundation and ensure every detail below is implemented.

**Shebang and guards:**
- `#!/usr/bin/env bash` (not `#!/bin/bash` — macOS bash 3 issue)
- `set -euo pipefail`
- Bash 4+ version guard: check `BASH_VERSINFO[0]` >= 4, exit with clear error if not

**Source oid_map.sh:**
- Use `SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"` then `source "${SCRIPT_DIR}/oid_map.sh"`
- Add sanity check after sourcing: verify `OID_MAP[T1_P1.E.1.oid]` is non-empty, exit with error if oid_map.sh failed to load

**Constants:**
- `SIM_URL="http://localhost:8080"`
- `RED='\033[0;31m'` and `NC='\033[0m'`
- `error()` helper function: prints red error to stderr and exits 1

**Port-forward lifecycle:**
- `PF_PID=""` variable
- `cleanup()` function that kills `$PF_PID` with `2>/dev/null || true`
- `trap cleanup EXIT`
- `kubectl port-forward svc/e2e-simulator 8080:8080 -n simetra &>/dev/null &`
- Capture `PF_PID=$!`
- `sleep 2` to allow port-forward to establish

**Argument parsing:**
- Check for `-v` as first arg, set `VERBOSE=1` and `shift` if present
- Exactly 1 remaining argument required, else error with usage hint
- Parse pattern with bash regex: `^([A-Z0-9_]+)-([VS])-([0-9]+)E-([0-9]+)R$`
- Extract TENANT, MODE, E_COUNT, R_COUNT from BASH_REMATCH
- On regex fail: error with "Invalid pattern" message showing expected format (CMD-06)

**Validation (in this order):**
1. Tenant validation: iterate `$VALID_TENANTS`, error if not found listing all valid tenants (CMD-04)
2. Count validation: check `E_COUNT <= TENANT_EVAL_COUNT[$TENANT]` and `R_COUNT <= TENANT_RES_COUNT[$TENANT]`, error with specific limit if exceeded (CMD-05)

**HTTP call helper `sim_post()`:**
- Takes oid and value arguments
- If VERBOSE, print `[verbose] POST ${SIM_URL}/oid/${oid}/${value}` to stderr
- `curl -sf -o /dev/null -w '%{http_code}' -X POST "${SIM_URL}/oid/${oid}/${value}"` with `2>/dev/null`
- Check curl exit code (|| error "curl failed for OID ${oid}")
- Check http_code == "200" (|| error "HTTP ${http_code} setting OID ${oid}=${value}")

**Stale helper `sim_stale()`:**
- Takes oid argument
- If VERBOSE, print `[verbose] POST ${SIM_URL}/oid/${oid}/stale` to stderr
- `curl -sf -o /dev/null -w '%{http_code}' -X POST "${SIM_URL}/oid/${oid}/stale"` with `2>/dev/null`
- Check curl exit code and http_code same as sim_post

**Translation loop `apply_role()`:**
- Takes role (E/R), count, and total arguments
- Iterates from 1 to total with `seq`
- For slots 1..count:
  - V mode: `sim_post "$oid" "${OID_MAP[${TENANT}.${role}.${i}.violated]}"`
  - S mode: `sim_stale "$oid"`
- For slots (count+1)..total (CMD-07 healthy reset):
  - Both V and S mode: `sim_post "$oid" "${OID_MAP[${TENANT}.${role}.${i}.healthy]}"`

**Main execution:**
- Call `apply_role "E" "$E_COUNT" "$e_total"`
- Call `apply_role "R" "$R_COUNT" "$r_total"`
- No output on success (silent = success convention)

**Make executable:** `chmod +x tests/e2e/lib/sim_command.sh`

**Script header comment:** Include usage line: `# Usage: ./sim_command.sh [-v] {Tenant}-{V/S}-{#}E-{#}R`
