This task runs kubectl and curl commands against the live K8s cluster. No code files are created or modified.

**Prerequisites:**
- Ensure kubectl port-forward to Prometheus is running: `kubectl port-forward -n simetra svc/prometheus 9090:9090 &`
- Wait 3 seconds for port-forward readiness

**Scenario 0: Baseline verification**
Confirm existing metrics are flowing before any changes:
```bash
curl -s -G "http://localhost:9090/api/v1/query" \
  --data-urlencode 'query=snmp_gauge{device_name="OBP-01",source="poll"}' | \
  jq '.data.result | length'
```
Expected: number > 0. Repeat for NPB-01. If 0, stop -- cluster is not healthy.

**Scenario 1: Add new OID**
1. Add test OID entry to simetra-oidmaps:
```bash
kubectl get cm simetra-oidmaps -n simetra -o json | \
  jq '.data["oidmaps.json"] |= (fromjson | . + {"1.3.6.1.4.1.47477.10.99.1.0": "test_oid_added"} | tojson)' | \
  kubectl apply -f -
```
2. Wait 10 seconds for watch event + processing
3. Check pod logs for reload message:
```bash
kubectl logs -n simetra -l app=snmp-collector --since=1m | grep -c "OidMap hot-reloaded"
```
Expected: 3 (one per replica). Each log should show `+1 added`.
4. Check logs for the specific addition:
```bash
kubectl logs -n simetra -l app=snmp-collector --since=1m | grep "test_oid_added"
```
Expected: Lines containing `OidMap added: 1.3.6.1.4.1.47477.10.99.1.0 -> test_oid_added`

**Scenario 2: Rename existing OID metric**
1. Rename obp_link_state_L1 to obp_link_state_L1_renamed:
```bash
kubectl get cm simetra-oidmaps -n simetra -o json | \
  jq '.data["oidmaps.json"] |= (fromjson | .["1.3.6.1.4.1.47477.10.21.1.3.1.0"] = "obp_link_state_L1_renamed" | tojson)' | \
  kubectl apply -f -
```
2. Wait 10 seconds
3. Check logs for change message:
```bash
kubectl logs -n simetra -l app=snmp-collector --since=1m | grep "obp_link_state_L1.*renamed"
```
Expected: Lines showing `OidMap changed: 1.3.6.1.4.1.47477.10.21.1.3.1.0 obp_link_state_L1 -> obp_link_state_L1_renamed`
4. Wait 30 seconds for Prometheus scrape cycle, then verify new metric name appears:
```bash
curl -s -G "http://localhost:9090/api/v1/query" \
  --data-urlencode 'query=snmp_gauge{metric_name="obp_link_state_L1_renamed",device_name="OBP-01"}' | \
  jq '.data.result | length'
```
Expected: > 0. Poll with 15s interval, 60s timeout if needed.

Record pass/fail for each scenario. Continue to Task 2.
