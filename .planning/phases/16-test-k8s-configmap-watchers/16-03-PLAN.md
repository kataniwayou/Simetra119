Before waiting 30+ minutes, check if reconnection has already occurred. The cluster has been running since deployment, and K8s watch API connections typically close every ~30 minutes.

```bash

```

2. If reconnection evidence found, check the timestamps:
```bash

```

3. If reconnection evidence is found, verify that a ConfigMap change AFTER the most recent reconnection still works. Apply a trivial change to simetra-oidmaps:
```bash

  kubectl apply -f -
```
4. Wait 10 seconds
5. Verify reload logged:
```bash
kubectl logs -n simetra -l app=snmp-collector --since=1m | grep "test_reconnect_verify"
```

6. Restore original ConfigMap:
```bash
kubectl apply -f deploy/k8s/snmp-collector/configmap.yaml
```
7. Wait 10 seconds and verify restore:
```bash

```
