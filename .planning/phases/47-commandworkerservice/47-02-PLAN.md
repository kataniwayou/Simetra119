  <action>
    Create `CommandWorkerService.cs` in `src/SnmpCollector/Services/`. This BackgroundService mirrors ChannelConsumerService almost line-for-line but executes SET commands instead of dispatching trap varbinds.

    **Constructor dependencies:**
    - `ICommandChannel` — channel to drain
    - `ISnmpClient` — for SetAsync calls
    - `ISender` — for dispatching response varbinds through MediatR pipeline
    - `IDeviceRegistry` — for community string lookup at execution time
    - `ICommandMapService` — for OID resolution (ResolveCommandOid) and reverse name resolution (ResolveCommandName)
    - `ICorrelationService` — for OperationCorrelationId (same as ChannelConsumerService)
    - `PipelineMetricService` — for IncrementCommandSent/IncrementCommandFailed counters
    - `IOptions<SnapshotJobOptions>` — for IntervalSeconds * TimeoutMultiplier timeout calculation
    - `ILogger<CommandWorkerService>`

    **ExecuteAsync (mirrors ChannelConsumerService lines 49-84):**
    1. Log Information: "Command channel worker started"
    2. `await foreach (var req in _commandChannel.Reader.ReadAllAsync(stoppingToken))`
    3. Set `_correlation.OperationCorrelationId = _correlation.CurrentCorrelationId` before each item
    4. Call `ExecuteCommandAsync(req, stoppingToken)` in try block
    5. Catch `OperationCanceledException when (stoppingToken.IsCancellationRequested)` — break
    6. Catch `Exception` — log Warning with CommandName and DeviceName, increment `IncrementCommandFailed(req.DeviceName)`, continue
    7. After loop: Log Information: "Command channel worker completed"

    **ExecuteCommandAsync (private async method):**
    1. **Resolve OID**: `var oid = _commandMapService.ResolveCommandOid(req.CommandName)`. If null: log Warning "Command {CommandName} not found in command map for {DeviceName} -- skipping", increment CommandFailed, return.
    2. **Resolve device**: `_deviceRegistry.TryGetByIpPort(req.Ip, req.Port, out var device)`. If false: log Warning "Device {Ip}:{Port} not found in registry for {DeviceName} -- skipping", increment CommandFailed, return.
    3. **Build Variable**: `var snmpData = SharpSnmpClient.ParseSnmpData(req.Value, req.ValueType)` (static call is fine — parsing is not protocol behavior). `var variable = new Variable(new ObjectIdentifier(oid), snmpData)`.
    4. **SET with timeout** (mirrors MetricPollJob line 93): Create linked CTS from stoppingToken, `CancelAfter(TimeSpan.FromSeconds(intervalSeconds * timeoutMultiplier))`. Build `IPEndPoint` from `device.ResolvedIp` and `device.Port`. Build `OctetString` community from `device.CommunityString`. Call `_snmpClient.SetAsync(VersionCode.V2, endpoint, community, variable, timeoutCts.Token)`.
    5. **Handle timeout**: Catch `OperationCanceledException when (!stoppingToken.IsCancellationRequested)` — this is a SET timeout (not shutdown). Log Warning "Command {CommandName} timed out for {DeviceName}", increment CommandFailed, return. Let `OperationCanceledException` when stoppingToken IS cancelled propagate up to the outer loop handler.
    6. **Dispatch response varbinds**: For each varbind in response, call `_commandMapService.ResolveCommandName(varbind.Id.ToString())` to get metricName (may be null). Create `SnmpOidReceived` with: Oid=varbind.Id.ToString(), AgentIp=IPAddress.Parse(device.ResolvedIp), DeviceName=req.DeviceName (NOT device.Name — CONTEXT.md locked decision), Value=varbind.Data, Source=SnmpSource.Command, TypeCode=varbind.Data.TypeCode, MetricName=metricName. Dispatch via `_sender.Send(msg, stoppingToken)`.
    7. **Increment success**: `_pipelineMetrics.IncrementCommandSent(req.DeviceName)` after all varbinds dispatched.

    CRITICAL anti-patterns from CONTEXT.md and RESEARCH.md:
    - Do NOT use `device.Name` for DeviceName on SnmpOidReceived — use `req.DeviceName`
    - Do NOT call `_commandChannel.Writer.Complete()` anywhere
    - Do NOT drain remaining commands on shutdown
    - The timeout catch must distinguish SET timeout from host shutdown using `!stoppingToken.IsCancellationRequested`
