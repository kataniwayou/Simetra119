```
# Scenario 113: TVM-07 Partial Percent -- 50% evaluate and resolved percentages via partial violations
# Uses tenant-cfg11-pss-partial.yaml (e2e-pss-partial: 2 eval + 2 resolved holders)
# OIDs: 5.1 = eval1 (Min:10), 6.1 = eval2 (Min:10), 5.2 = res1 (Min:1), 6.2 = res2 (Min:1)

#   TVM-07C: tenant_metric_resolved_percent == 50 (1 of 2 resolved holders violated)
#   TVM-07D: tenant_evaluation_duration_milliseconds_count delta > 0 (evaluation ran)
#   TVM-07E: tenant_command_dispatched_percent > 0 (commands dispatched on Unresolved path)
```

Script flow (follow 110/111 pattern exactly):

1. **Setup:** Set FIXTURES_DIR. Save original tenant ConfigMap via `save_configmap`. Apply `tenant-cfg11-pss-partial.yaml` via kubectl. Wait for tenant vector reload via `poll_until_log 60 5`.

2. **Prime all 4 OIDs in-range** for readiness grace:
   - `sim_set_oid "5.1" "10"` (eval1 in-range)
   - `sim_set_oid "6.1" "10"` (eval2 in-range)
   - `sim_set_oid "5.2" "1"` (res1 in-range)
   - `sim_set_oid "6.2" "1"` (res2 in-range)
   - Sleep 8s for grace (6s grace + 2s margin)

3. **Snapshot duration baseline:**
   `BEFORE_DURATION=$(snapshot_counter "tenant_evaluation_duration_milliseconds_count" 'tenant_id="e2e-pss-partial",priority="1"')`

4. **Trigger partial violations:**
   - `sim_set_oid "5.1" "0"` (eval1 violated, < Min:10)
   - `sim_set_oid "5.2" "0"` (res1 violated, < Min:1)
   - eval2 (6.1) stays at 10 (in-range), res2 (6.2) stays at 1 (in-range)

5. **Wait for tier=4:** `poll_until_log 30 1 "e2e-pss-partial.*tier=4" 15`. Then sleep 15s for Prometheus scrape.

6. **Sub-assertions:**

   **TVM-07A** (state=3 Unresolved):
   Query `tenant_evaluation_state{tenant_id="e2e-pss-partial"}`, parse with jq, cut -d. -f1. Assert == 3.

   **TVM-07B** (evaluate_percent == 50):
   Query `tenant_metric_evaluate_percent{tenant_id="e2e-pss-partial"}`, parse value. Use awk to assert value == 50 (integer compare after float parse: `awk '{exit (int($1+0.5) == 50) ? 0 : 1}'`). This is 1 of 2 evaluate holders violated.

   **TVM-07C** (resolved_percent == 50):
   Query `tenant_metric_resolved_percent{tenant_id="e2e-pss-partial"}`, parse value. Same awk integer compare for == 50. This is 1 of 2 resolved holders violated.

   **TVM-07D** (duration delta > 0):
   Snapshot AFTER_DURATION, compute DELTA_DURATION. Use `assert_delta_gt`.

   **TVM-07E** (dispatched_percent > 0):
   Query `tenant_command_dispatched_percent{tenant_id="e2e-pss-partial"}`, check series count > 0 (same pattern as TVM-05B in scenario 111).

7. **Cleanup:** `reset_oid_overrides`, then restore original tenant ConfigMap from snapshot.
