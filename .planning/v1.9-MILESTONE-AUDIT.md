## Observations

**Mutation safety in ValidateAndBuildTenants:** The metric.Threshold = null mutation at TenantVectorWatcherService.cs line 167 operates on the same object reference added to cleanMetrics at line 221. The null is visible when Reload processes the clean options. This is correct because the watcher deserializes a fresh TenantVectorOptions graph from JSON on each reload cycle.

**Constructor parameter ordering:** MetricSlotHolder signature (ip, port, metricName, intervalSeconds, timeSeriesSize=1, graceMultiplier=2.0, threshold=null) places all optional params last with non-nullable optional (graceMultiplier) before nullable optional (threshold). The heartbeat holder at TenantVectorRegistry.cs line 76 uses only 4 positional arguments, correctly taking all three optional param defaults.

**Absence of runtime evaluation:** This milestone is intentionally structural-only. No code in the pipeline reads holder.Threshold to make pass/fail decisions. The field exists, is stored correctly, and is ready for a future evaluation phase.
