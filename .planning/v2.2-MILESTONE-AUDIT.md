## Tech Debt

### Phase 63: run-stage2.sh

1. **Stale filenames (DEFECT):** Lines 91-93 reference `53-pss-01-readiness.sh`, `54-pss-02-resolved.sh`, `55-pss-03-healthy.sh` — these were renamed during Phase 62 development. The `if [ -f "$scenario" ]` guard silently skips them, so only 3 of 6 Stage 1 scenarios run in standalone `run-stage2.sh` mode. `run-stage3.sh` has the correct names.

2. **No cleanup trap:** `run-stage2.sh` lacks a cleanup trap for ConfigMap restore on crash. Individual scenarios use `|| true` on restore calls, which is a weaker guarantee.

### Phase 64: Test Timing

3. **PSS-18c/PSS-19c flakiness:** The 10s sleep + 12s `--since` window creates a 2s overlap where prior-cycle G2 tier logs can bleed in. The dual-proof `d` variants (metric non-increment) cover the same assertion deterministically. Fix: increase `--since` to match sleep, or increase stabilization wait.

4. **Report category cosmetic gap:** Standalone `run-stage3.sh` generates a report where the PSS category (indices 52-67) exceeds the array length (only 16 entries). The category is silently skipped. Only affects standalone runner output formatting, not `run-all.sh`.

### Total: 4 items across 2 phases
