## Bugs Found

### BUG-1 (P1): MLC-03 scenario queries wrong resolved_name -- test will always fail

File: tests/e2e/scenarios/96-mlc03-source-command.sh
Lines: 4 (comment), 90, 104, 111, 114

Scenario 96 uses resolved_name=e2e_set_bypass in all Prometheus queries. This is
the command name, not the metric name. The actual resolved_name for OID .999.4.4.0
is e2e_command_response.

Tracing confirms:
1. deploy/k8s/snmp-collector/simetra-oid-command-map.yaml maps CommandName=e2e_set_bypass
   to OID 1.3.6.1.4.1.47477.999.4.4.0
2. tests/e2e/fixtures/.original-oid-metric-map-configmap.yaml maps that OID to
   MetricName=e2e_command_response
3. MetricName becomes the resolved_name label in Prometheus

Wrong identifier on:
- Line 90: snmp_gauge with resolved_name=e2e_set_bypass returns 0 results always
- Line 104: success log uses wrong name
- Line 111: evidence query uses wrong name
- Line 114: failure message references wrong name

Required fix: Replace resolved_name=e2e_set_bypass with resolved_name=e2e_command_response
on lines 4, 90, 104, 111, 114.

### BUG-2 (P2): MCV-12 wrong start_port_forward signature -- simulator port-forward not restored

File: tests/e2e/scenarios/81-mcv12-poll-recovered.sh
Line: 31

After scaling the simulator back up to 1 replica (lines 14-15), the scenario calls:

    start_port_forward simetra e2e-simulator 8080:8080 2>/dev/null || true

The start_port_forward function in tests/e2e/lib/kubectl.sh has signature:

    start_port_forward SERVICE_NAME LOCAL_PORT_INT REMOTE_PORT_INT

The call passes simetra as service name (kubectl tries svc/simetra, does not exist),
e2e-simulator as local port (non-numeric), and 8080:8080 as remote port.
The || true suppresses the error silently.

Impact: After scenario 81, sim_set_oid calls hit http://localhost:8080 with no listener.
Scenarios 83 (CCV-01), 84 (CCV-02/03), 85 (CCV-04), and 93 (MLC-01) all use sim_set_oid.
With set -euo pipefail in run-all.sh, the first failing sim_set_oid without || true
aborts the entire suite.

Required fix: Replace line 31 with:

    start_port_forward e2e-simulator 8080 8080

## Verified Correct

### 1. sort -V in run-all.sh

tests/e2e/run-all.sh uses ls -1 ... | sort -V (version sort). Correct ordering confirmed:
sort -V gives 99 < 100 < 101; plain sort gives 100 < 101 < 99. All 38 scenario files
are sourced in correct numeric order.

### 2. assert_delta_eq and assert_delta_ge in common.sh

Both helpers added by phase 66 call exactly one of record_pass/record_fail per invocation
(1 SCENARIO_RESULTS entry per call). Consistent with assert_delta_gt convention.

### 3. Scale-down approach in scenarios 80-81

Scenario 80: kubectl scale deployment e2e-simulator -n simetra --replicas=0 (no ConfigMap
mutation, no DynamicPollScheduler jitter). Scenario 81: --replicas=1. Correct per phase 68.

### 4. CCV-03 relaxed assertion in scenario 84

84-ccv02-03-command-suppressed.sh checks DELTA_SUPP_W2 > 0 (phase 68-03 relaxed assertion)
rather than DELTA_SENT_W2 == 0, avoiding the race condition.

### 5. CCV-04 timeout path in scenario 85

85-ccv04-command-failed.sh uses FAKE-UNREACHABLE device with dual ConfigMap save/restore
(simetra-devices and simetra-tenants). Correct per phase 68-04.

### 6. ConfigMap save/restore pattern

Scenarios 85, 93, 96 all save ConfigMaps before mutation and restore at end.
Scenario 96 uses if-f guard for graceful handling of missing snapshot files.

### 7. OID metric map fixture correctness

.original-oid-metric-map-configmap.yaml correctly maps OID .999.4.4.0 to
MetricName=e2e_command_response. Scenario 93 (MLC-01) uses the correct resolved_name.
Only scenario 96 (MLC-03) uses the wrong name.

### 8. sim.sh port-forward dependency

run-all.sh establishes localhost:8080 port-forward at startup. Scenario 81 is
responsible for re-establishing after scale-up but silently fails (BUG-2).
