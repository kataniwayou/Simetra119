# Milestone v2.4: Tenant Vector Metrics

**Status:** ✅ SHIPPED 2026-03-23
**Phases:** 72-75

## Overview

Expose per-tenant internal evaluation state from SnapshotJob as 8 OTel instruments (6 counters, 1 gauge, 1 histogram) on a third meter that exports on all instances, and surface them in a new operations dashboard table with per-pod tenant status.

## Milestone Summary

**Key Accomplishments:**
1. TenantMetricService — 8 OTel instruments on SnmpCollector.Tenant meter, exporting on all instances (not leader-gated)
2. EvaluateTenant instrumentation — counting helpers, RecordAndReturn pattern, per-cycle batched metrics at exit
3. Operations dashboard Tenant Status table — 13 columns with state color mapping, trend arrows, per-cycle integer counters
4. 6 E2E scenarios (107-112) proving full instrument→Prometheus→dashboard pipeline across all evaluation paths
5. All-instances export verified — follower pods export tenant metrics while snmp_gauge remains leader-gated

**Key Decisions:**
- Counter queries use increase()[30s] not rate() — operators want per-cycle integer counts
- PromQL zero-fallback via `or on(...) (tenant_state != 0) * 0` for non-NotReady tenants
- PromQL column removed from dashboard (user decision during discuss-phase)
- Grafana switched to manual dashboard management (removed file provisioner)
- 5 redundant ops panels removed (Tenant Vector Routed, Snapshot Cycle Duration, Command Dispatched/Failed/Suppressed)
- All tenant metrics batched at EvaluateTenant exit point (not interleaved with dispatch loop)

**Issues Resolved:**
- Dashboard duplicate rows caused by label set mismatch in merge transform (fixed with max by() wrapper)
- Counter columns showing blank for evaluated tenants (fixed with zero-fallback PromQL)
- E2E NotReady scenario timing (CopyFrom carries prior series; softened to informational assertion)
- ROADMAP typo: "tenant_gauge_duration_milliseconds" → correct name is "tenant_evaluation_duration_milliseconds"

**Technical Debt:**
- NotReady E2E assertion softened due to CopyFrom behavior — unit tests provide isolation guarantee
- `tenant_command_failed_total` existence never verified in E2E isolation (requires SET failure scenario)
- `resource_to_telemetry_conversion.enabled` external dependency undocumented in code
