## Milestone Summary

**Key Decisions:**
- snmp.event.rejected fires only on ValidationBehavior failures (not unmapped OIDs)
- dispatched and suppressed are independent counters (not mutually exclusive)
- Unmapped CommandName causes tenant skip at load time (validation-time, not runtime)
- CCV-04 uses timeout path via unreachable IP for command.failed
- MCV-11/12 use simulator scale-down (not FAKE-UNREACHABLE ConfigMap)
- MNP-01 reframed: heartbeat IS in snmp_gauge (via MergeWithHeartbeatSeed), test snmp_info absence
- sort -V required in run-all.sh for 100+ scenario ordering
- CommandWorkerService resolves SET response names via command map (not OID map)

**Issues Resolved:**
- CCV-03 assertion: dispatched fires on every tier=4 (correct by design)
- CCV-04 trigger: unmapped CommandName → tenant skip (switched to timeout path)
- MLC-03 resolved_name: e2e_set_bypass (command map), not e2e_command_response (OID map)
- Scenario 81 port-forward: wrong call signature fixed
- sort -V: bash glob sorts 100- before 11-, fixed with sort -V pipe

**Technical Debt:**
- Report category indices assume 1 SCENARIO_RESULTS entry per scenario file (CCV scenarios produce 2-3). Display-only issue.
