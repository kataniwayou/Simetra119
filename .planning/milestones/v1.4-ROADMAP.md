# Milestone v1.4: E2E System Verification

**Status:** SHIPPED 2026-03-09
**Phases:** 20-24
**Total Plans:** 11

## Overview

This milestone builds an E2E test harness that proves the full SNMP-to-Prometheus pipeline works correctly under normal operation, configuration mutations, and edge cases. A dedicated test simulator covers scenarios that existing OBP/NPB simulators cannot. A bash test runner orchestrates 27 test scenarios sequentially, querying Prometheus HTTP API and kubectl logs for pass/fail evidence. No SnmpCollector code is modified -- findings are documented only.

## Milestone Summary

**Key Decisions:**
- Bash E2E runner over pytest -- sufficient for sequential SNMP scenarios, no extra dependencies
- Poll-until-satisfied over fixed sleeps -- handles OTel 15s export interval variability
- Delta-based counter assertions -- correct for cumulative temporality metrics
- ConfigMap snapshot/restore -- enables safe mutation testing without manual cleanup
- Sourced-script pattern -- scenarios have no shebang, inherit lib functions from run-all.sh
- Pass-with-caveat for WATCH-04 -- watcher reconnection rarely observable in short test windows

**Issues Resolved:**
- Report category indices misaligned (33 runtime results vs 27 indexed slots) -- fixed in 24-03
- Scenario 12 stale metric name obp_link_state_ch1 vs obp_link_state_L1 -- fixed in 24-03
