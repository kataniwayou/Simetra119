# Milestone v2.6: E2E Manual Tenant Simulation Suite

**Status:** SHIPPED 2026-03-24
**Phases:** 82-83
**Total Plans:** 3

## Overview

An interactive command interpreter that lets a human operator drive any tenant into any violation state on demand — using a terse CLI pattern syntax — so tenant state transitions can be verified by watching Grafana without managing 17 script files.

## Phases

### Phase 82: Fixture & OID Mapping

**Goal**: A 4-tenant environment with collision-free OIDs is live in the cluster, all tenants start Healthy, and a hardcoded mapping file defines every OID suffix and value needed to violate or restore any tenant role.
**Depends on**: Phase 81 (v2.5 complete)
**Plans**: 2 plans

Plans:

- [x] 82-01-PLAN.md — Register 24 v2.6 OIDs in simulator, OID metric map, and device poll config
- [x] 82-02-PLAN.md — Create 4-tenant fixture ConfigMap and OID mapping file

**Details:**
- 24 new OIDs across subtrees 8-11 (one per tenant, collision-free by construction)
- OID metric map extended (K8s + local dev)
- Per-tenant poll groups at IntervalSeconds=5
- Tenant fixture: tenant-cfg12-v26-four-tenant.yaml with T1_P1/T2_P1 (P1) and T1_P2/T2_P2 (P2)
- OID mapping file: tests/e2e/lib/oid_map.sh (72 entries, associative array)
- Simulator baseline values set to healthy (eval=10, res=1)

### Phase 83: Command Interpreter

**Goal**: A command interpreter accepts `{Tenant}-{V/S}-{#}E-{#}R` patterns from the Claude Code CLI, validates them against the mapping, translates them to simulator HTTP API calls, and produces clear errors for invalid input.
**Depends on**: Phase 82
**Plans**: 1 plan

Plans:

- [x] 83-01-PLAN.md — Create standalone sim_command.sh command interpreter

**Details:**
- Standalone Bash script at tests/e2e/lib/sim_command.sh
- Pattern parsing: {Tenant}-{V/S}-{#}E-{#}R with regex validation
- 3 error types: unknown tenant, count exceeded, malformed pattern (all red with hints)
- V-mode: violated/healthy value POSTs to /oid/{suffix}/{value}
- S-mode: stale POSTs to /oid/{suffix}/stale
- Stateless: every command is a full state declaration, non-affected slots reset to healthy
- Self-managed kubectl port-forward with trap cleanup
- Silent on success, -v flag for verbose

## Milestone Summary

**Fixes Applied During UAT:**

- Simulator: return noSuchInstance as value (not raise NoSuchInstanceError) — was aborting entire multi-varbind GET
- Poll groups: split from 1 shared → 4 per-tenant — cross-tenant staleness via timing jitter
- Poll interval: 1s → 5s — jitter-induced Unresolved blips
- Baseline values: 0 → healthy (eval=10, res=1) — tenants were starting Violated

**Quick Tasks:**

- 090: Dashboard table footer row counts
- 091: SnapshotJob evaluation pipeline logging (state, transitions, gate blocks)

**Key Decisions:**

- Standalone script (not sourced function) — invoked directly from CLI
- Port-forward managed inside script — operator needs no kubectl knowledge
- Silent on success (Unix convention) — red errors with helpful hints
- One pattern per invocation, no special commands (RESET, STATUS)
- Stateless — each command fully declares tenant state
- Per-tenant poll groups at 5s interval — isolation + stability

**Issues Resolved:**

- pysnmp NoSuchInstanceError aborting multi-varbind GET (simulator fix)
- Cross-tenant staleness from shared poll group (device config split)
- Missing SnapshotJob logging for state transitions, gate blocks, stale detection

**Issues Deferred:**

- SnapshotJob → CommandWorkerService correlation gap (CommandRequest carries no correlationId)
- Prometheus stale series from terminated pods after rolling restart (~5 min expiry)

**Technical Debt Incurred:**

None.
