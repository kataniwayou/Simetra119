## v2.3 Requirements

### Pipeline Counter Validity

- [x] **MCV-01**: `snmp.event.published` increments exactly once per OID entering the pipeline (poll path)
- [x] **MCV-02**: `snmp.event.published` increments exactly once per OID entering the pipeline (trap path)
- [x] **MCV-03**: `snmp.event.handled` increments only for mapped OIDs that reach the terminal handler
- [x] **MCV-04**: `snmp.event.handled` does NOT increment for unmapped/rejected OIDs
- [x] **MCV-05**: `snmp.event.rejected` stays 0 during normal operation (fires only on ValidationBehavior failures, not unmapped OIDs)
- [x] **MCV-06**: `snmp.event.rejected` does NOT increment for mapped OIDs
- [x] **MCV-07**: `snmp.event.errors` stays 0 during a full normal E2E run (safety-net counter)
- [x] **MCV-08**: `snmp.poll.executed` increments once per poll cycle regardless of success/failure
- [x] **MCV-09**: `snmp.trap.received` increments for valid community traps only
- [x] **MCV-10**: `snmp.trap.auth_failed` increments for bad community traps and does NOT increment `trap.received`
- [x] **MCV-11**: `snmp.poll.unreachable` increments after simulator scale-down (3 consecutive failures)
- [x] **MCV-12**: `snmp.poll.recovered` increments when simulator scaled back up
- [x] **MCV-13**: `snmp.tenantvector.routed` increments when tenant vector fan-out write succeeds

### Command Counter Validity

- [x] **CCV-01**: `snmp.command.dispatched` increments when SnapshotJob enqueues a command at tier=4
- [x] **CCV-02**: `snmp.command.suppressed` increments on repeated tier=4 within suppression window
- [x] **CCV-03**: `snmp.command.dispatched` fires on every tier=4 (independent of suppression — dispatched = decision, suppressed = prevention)
- [x] **CCV-04**: `snmp.command.failed` increments on SET timeout to unreachable device

### Business Metric Value Correctness

- [x] **MVC-01**: `snmp_gauge` value matches simulator Gauge32 value (42)
- [x] **MVC-02**: `snmp_gauge` value matches simulator Integer32 value (100)
- [x] **MVC-03**: `snmp_gauge` value matches simulator Counter32 raw value (5000)
- [x] **MVC-04**: `snmp_gauge` value matches simulator Counter64 raw value (1000000)
- [x] **MVC-05**: `snmp_gauge` value matches simulator TimeTicks value (360000)
- [x] **MVC-06**: `snmp_info` value label matches simulator OctetString value ("E2E-TEST-VALUE")
- [x] **MVC-07**: `snmp_info` value label matches simulator IpAddress value ("10.0.0.1")
- [x] **MVC-08**: `snmp_gauge` value updates when simulator value changes (42→99)

### Label Correctness

- [x] **MLC-01**: `source="poll"` for polled OIDs
- [x] **MLC-02**: `source="trap"` for trap-originated OIDs
- [x] **MLC-03**: `source="command"` for SET response OIDs (resolved_name=e2e_set_bypass via command map)
- [x] **MLC-04**: `source="synthetic"` for aggregated metrics
- [x] **MLC-05**: `snmp_type` label matches SNMP type (gauge32, integer32, counter32, counter64, timeticks)
- [x] **MLC-06**: `snmp_type` label matches SNMP type (octetstring, ipaddress)
- [x] **MLC-07**: `resolved_name` label matches oidmaps.json mapping
- [x] **MLC-08**: `device_name` label matches community-derived name

### Negative Proofs

- [x] **MNP-01**: Heartbeat never in snmp_info + never resolved_name="Unknown" (heartbeat IS in snmp_gauge as "Heartbeat")
- [x] **MNP-02**: Unmapped OID produces no snmp_gauge or snmp_info
- [x] **MNP-03**: Bad community trap produces no snmp_gauge or snmp_info
- [x] **MNP-04**: `snmp.trap.dropped` stays 0 during normal E2E run
- [x] **MNP-05**: Follower pod exports no snmp_gauge/snmp_info (via k8s_pod_name label)
