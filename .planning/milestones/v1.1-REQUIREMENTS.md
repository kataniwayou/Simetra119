## v1.1 Requirements

### OID Map Structure

- [x] **OIDM-01**: OID map naming convention uses device-type prefix + metric + index suffix (e.g., `obp_link_state_L1`, `npb_port_rx_octets_P1`)
- [x] **OIDM-02**: OID map structure decision — single shared oidmaps.json with both device types (decided during Phase 11, later simplified in v1.2 to bare dict)
- [x] **OIDM-03**: OID map populated for OBP device — 4 links with 24 OIDs (state, channel, optical power R1-R4)
- [x] **OIDM-04**: OID map populated for NPB device — 8 ports with 68 OIDs (system health, per-port traffic, port status)

### Simulator

- [x] **SIM-01**: OBP simulator serves 24 OIDs with power random walk on R1-R4 receivers
- [x] **SIM-02**: OBP simulator sends StateChange traps for all 4 links with channel varbind
- [x] **SIM-03**: NPB simulator serves 68 OIDs with Counter64 traffic profiles and OctetString system health
- [x] **SIM-04**: NPB simulator sends portLinkChange traps for 6 active ports (P4/P8 permanently down)
- [x] **SIM-05**: Both simulators use `Simetra.{DeviceName}` community string convention (configurable via DEVICE_NAME env var)
- [x] **SIM-06**: Simulator K8s deployments with pysnmp health probes and DEVICE_NAME env vars

### Device Documentation

- [x] **DOC-01**: OBP OID documentation — each OID with JSONC comments (SNMP type, units, range)
- [x] **DOC-02**: NPB OID documentation — each OID with JSONC comments (SNMP type, units, range)

### MetricPoll Configuration

- [x] **POLL-01**: devices.json ConfigMap with OBP-01 MetricPoll (24 OIDs, 10s interval, K8s DNS address)
- [x] **POLL-02**: devices.json ConfigMap with NPB-01 MetricPoll (68 OIDs, 10s interval, K8s DNS address)
