## Milestone Summary

**Key Decisions:**
- Naming convention: `obp_{metric}_L{n}` for OBP, `npb_{metric}` / `npb_port_{metric}_P{n}` for NPB
- NPB OID tree: `47477.100.1.{id}.0` for system, `47477.100.2.{port}.{id}.0` for per-port
- Single MetricPoll per device with all OIDs (simpler config, shared 10s interval)
- pysnmp-based K8s health probes (more maintainable than raw ASN.1 hex packets)
- DNS resolution fallback in DeviceRegistry for K8s Service names
- Optional CommunityString override on DeviceOptions (null falls back to Simetra.{Name} convention)

**Issues Resolved:**
- Config binding mismatch (oidmap-obp.json missing OidMap wrapper) — fixed in Plan 11-03
- Headless service label mismatch (app:simetra vs app:snmp-collector) — fixed in quick-019

**Technical Debt Incurred:**
- verify-e2e.sh check 4 queries snmp_gauge for npb_cpu_util (OctetString routes to snmp_info)
- Production simetra-devices template has wrong MetricPolls schema (objects vs strings)
- Dev K8s missing simetra-oidmaps and simetra-devices ConfigMap definitions
