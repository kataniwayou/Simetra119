## Milestone Summary

**Key Decisions:**
- Heartbeat is infrastructure, not a tenant metric — removed from tenant vector entirely
- "Simetra" not in DeviceRegistry → TryGetDeviceByName returns false → fan-out naturally skipped (zero special cases)
- Two liveness layers: job completion (all jobs) + pipeline arrival (heartbeat only)
- IHeartbeatLivenessService uses volatile long (UTC ticks) for lock-free stamping
- Staleness window: HeartbeatJobOptions.IntervalSeconds × 2.0 = 30s
- null LastArrival (startup) = always stale — K8s failureThreshold provides margin

**Issues Resolved:**
- Hardcoded heartbeat tenant removed (-115 lines of special cases)

**Issues Deferred:**
- Runtime threshold evaluation (separate milestone)
- Metric staleness detection for business metrics (uses IntervalSeconds × GraceMultiplier)

**Technical Debt Incurred:**
