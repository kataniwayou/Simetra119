## E2E Flow Verification

### Flow 1: Healthy Baseline (STS-01) -- 29-sts-01-healthy.sh

1. kubectl apply tenant-cfg01-single.yaml (e2e-tenant-A, Evaluate Max:80, Resolved Min:1.0)
2. sim_set_scenario healthy: .4.1=5 (< Max:80), .4.2=2, .4.3=2 (>= Min:1.0)
3. SnapshotJob: tier-1 not stale, tier-2 resolved in-range, tier-3 evaluate not violated, tier=3 fires
4. poll_until_log for "tier=3 — not all evaluate metrics violated" (UTF-8 em-dash U+2014 confirmed byte-identical to SnapshotJob.cs line 153)
5. DELTA_SENT == 0 and DELTA_SUPP == 0

**Status: COMPLETE**

### Flow 2: Command Dispatch (STS-02) -- 30-sts-02-evaluate-violated.sh

1. kubectl apply tenant-cfg01-single.yaml
2. sim_set_scenario command_trigger: .4.1=90 (> Max:80), .4.2=2, .4.3=2 (resolved in-range)
3. SnapshotJob: tier-1 passes, tier-2 passes, tier-3 violated, tier-4 fires
4. poll_until_log 90s for "tier=4 — commands enqueued" (SnapshotJob.cs line 191)
5. assert_delta_gt DELTA_SENT 0

90s timeout accommodates TimeSeriesSize fill: ~3 poll cycles + SnapshotJob 15s = ~75s minimum, 15s margin.

**Status: COMPLETE**

### Flow 3: Resolved Gate (STS-03) -- 31-sts-03-resolved-gate.sh

1. kubectl apply tenant-cfg01-single.yaml
2. sim_set_scenario default: .4.2=0, .4.3=0 (< Min:1.0, resolved violated)
3. SnapshotJob: tier-2 fires ConfirmedBad, halts before tier-4
4. poll_until_log for "tier=2 — all resolved violated, device confirmed bad" (SnapshotJob.cs line 139)
5. Zero delta on sent and suppressed counters
6. Negative assertion: kubectl logs --since=60s confirms no tier=4 lines

**Status: COMPLETE**

### Flow 4: Suppression Window (STS-04) -- 32-sts-04-suppression-window.sh

1. tenant-cfg01-suppression.yaml (SuppressionWindowSeconds=30, tenant ID e2e-tenant-A-supp)
2. W1: command_trigger -> poll_until_log tier=4 -> assert sent delta > 0 (32a, 32b)
3. W2: within 30s window, poll_until for snmp_command_suppressed_total{device_name="e2e-tenant-A-supp"} increment (32c, 32d). Suppressed counter label correctly uses tenant ID, matching IncrementCommandSuppressed(tenant.Id) in SnapshotJob.cs.
4. W3: sleep 20 expires window -> poll_until_log tier=4 -> assert sent delta > 0 (32e, 32f)

**Status: COMPLETE**

### Flow 5: Staleness Detection (STS-05) -- 33-sts-05-staleness.sh

1. tenant-cfg01-single.yaml
2. Priming: sim_set_scenario healthy + sleep 20 (required: HasStaleness returns false for null slots)
3. sim_set_scenario stale: .4.1=STALE, .4.2=STALE -- simulator raises NoSuchInstanceError
4. poll_until_log 90s for "tier=1 stale" (SnapshotJob.cs line 130)
5. Zero delta on sent and suppressed counters

**Status: COMPLETE**

### Flow 6: Multi-Tenant Parallel Same-Priority (MTS-01) -- 34-mts-01-same-priority.sh

1. tenant-cfg02-two-same-prio.yaml (e2e-tenant-A and e2e-tenant-B both Priority 1)
2. sim_set_scenario command_trigger
3. poll_until_log 90s for "e2e-tenant-A.*tier=4 — commands enqueued"
4. poll_until_log 60s for "e2e-tenant-B.*tier=4 — commands enqueued" (independent assertion)
5. assert_delta_gt DELTA 1 (aggregate snmp_command_sent_total{device_name="E2E-SIM"} delta >= 2)
6. All counter queries use sum() without pod label filter

**Status: COMPLETE**

### Flow 7: Advance Gate (MTS-02) -- 35-mts-02-advance-gate.sh

02A (gate blocked):
1. tenant-cfg03-two-diff-prio-mts.yaml (P1 Priority 1 SuppressionWindow=30s, P2 Priority 2)
2. sim_set_scenario command_trigger
3. poll_until_log 90s for "e2e-tenant-P1.*tier=4" (P1 Commanded, gate blocked)
4. Negative: iterate all pods, kubectl logs --since=120s, P2 tier log absent
5. Assert P1 sent delta > 0; sleep 18 + quiescence delta == 0

02B (gate passed):
6. No scenario reset; P1 suppressed at T=15s cycle (30s window > 15s interval), returns ConfirmedBad
7. poll_until_log 60s for "e2e-tenant-P2.*tier=4"
8. Total sent delta from original BEFORE_SENT baseline >= 2

**Status: COMPLETE**

### Flow 8: Aggregate via Synthetic Pipeline (ADV-01) -- 36-adv-01-aggregate-evaluate.sh

1. tenant-cfg04-aggregate.yaml (e2e-tenant-agg, Evaluate=e2e_total_util Max:80, TimeSeriesSize=3)
2. sim_set_scenario agg_breach: .4.5=50, .4.6=50 (sum=100 > Max:80); .4.2=2, .4.3=2 (resolved in-range)
3. Collector aggregate poll: sum(e2e_agg_source_a, e2e_agg_source_b) -> e2e_total_util=100 with source=synthetic label
4. SnapshotJob: tier-1 passes, tier-2 passes, tier-3 violated, tier-4 fires
5. poll_until_log 90s for "e2e-tenant-agg.*tier=4 — commands enqueued"
6. Assert sent delta > 0
7. sleep 30 -> query_prometheus snmp_gauge{resolved_name="e2e_total_util",source="synthetic"} -> count > 0
8. Recovery: sim_set_scenario healthy -> poll_until_log 90s for "e2e-tenant-agg.*tier=3"

**Status: COMPLETE**

### Flow 9: Time-Series Depth-3 All-Samples (ADV-02) -- 37-adv-02-depth3-allsamples.sh

Breach phase:
1. tenant-cfg04-aggregate.yaml (TimeSeriesSize=3, GraceMultiplier=2.0)
2. sim_set_scenario agg_breach: 3 poll cycles fill all slots with sum=100 > Max:80 (~75s minimum)
3. poll_until_log 90s for "e2e-tenant-agg.*tier=4 — commands enqueued" (37a)
4. Assert sent delta > 0 (37b)

Recovery phase:
5. sim_set_scenario healthy: AreAllEvaluateViolated fails (not all 3 slots violated) -> tier-3
6. Recovery counter baseline captured after scenario switch
7. poll_until_log 90 5 "e2e-tenant-agg.*tier=3" 30 (since=30 excludes pre-breach tier=3 logs from same tenant) (37c)
8. Assert recovery counter delta == 0 (37d)

**Status: COMPLETE**
